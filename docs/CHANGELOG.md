# История релизов

Этот документ фиксирует только те изменения, что влияют на пользователей и эксплуатацию системы:

- Конечных пользователей сайта
- Администраторов/контент‑менеджеров
- Инженеров эксплуатации (DevOps/SRE)

Основы и стиль:
- Формат — Keep a Changelog; версии — SemVer (MAJOR.MINOR.PATCH).
- Пиши кратко и в повелительном наклонении: «Добавь», «Обнови», «Исправь», «Удали».
- Для каждой записи обязательно указывай Why/Impact (зачем/эффект).
- Для значимых тем добавляй подпункт: `Files → Effects → Intent` (и при необходимости краткий QA/Checks).
- Игнорируй сугубо разработческие правки (рефакторинг без эффекта для пользователя, косметические перестановки).

Процесс:
- После каждой завершённой задачи обновляй раздел Unreleased (Definition of Done).
- При релизе переносите Unreleased в новый раздел версии с датой, создайте Git‑тег.
- Даты указывай в формате YYYY‑MM‑DD.
- Для потенциально несовместимых изменений явно помечай Breaking Change.

Секции изменений (Keep a Changelog):
- `Added` — новая функциональность.
- `Changed` — изменённое поведение без поломки совместимости.
- `Deprecated` — объявлено устаревшим (с планом удаления).
- `Removed` — удалено.
- `Fixed` — исправлено.
- `Security` — уязвимости/усиление защиты.

Метод `isBirthdayPeriod()` проверяет период в текущем году, а если день рождения уже прошел — в следующем году, что обеспечивает корректную работу в конце года.
	- Intent → Создать автоматическую систему скидок на день рождения для владельцев карт лояльности, которая будет применяться в период вокруг дня рождения пользователя. Это повышает лояльность клиентов и стимулирует покупки в праздничный период. Настраиваемая величина скидки позволяет гибко управлять маркетинговыми акциями, а поддержка внешних API обеспечивает единообразное применение скидок на сайте и в мобильном приложении.
