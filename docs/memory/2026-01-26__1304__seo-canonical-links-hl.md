# Task Memory Card
Date: 2026-01-26
Task Key: 1304
Title: Хайлоадблок канонических ссылок из поиска

## Context
- Добавление страниц популярных запросов поиска дало отличный эффект: большая часть индексируемых страниц теперь из поиска.
- Для расширения эффекта необходимо перенести вес этих страниц в каталог.
- Первый шаг — создание механизма замены канонических ссылок: на страницах поиска подставлять canonical на страницу каталога.
- Требовалось создать хайлоадблок SeoCanonicalLinks с полями UF_URL_FROM (исходный URL) и UF_URL_TO (канонический URL).

## Key Decisions
- Создать хайлоадблок SeoCanonicalLinks по аналогии с SeoRedirectsTable (для редиректов).
- Поля: UF_URL_FROM (обязательное, строка, поиск) и UF_URL_TO (обязательное, строка, поиск) — оба с размером 80 символов.
- Реализовать механизм замены в классе Seo: отдельный метод `getCanonicalLinkReplacement()` для проверки замены, интеграция в существующий `getCanonicalLink()`.
- Использовать кеширование запросов к хайлоадблоку (APP_CACHE_D) для производительности.
- Проверять оба варианта URL: с завершающим слэшем и без (rtrim($url, '/') . '/' и $url) для надежности.

  - Метод `getCanonicalLinkReplacement($url)` (строки 864-880): проверяет наличие записи в SeoCanonicalLinksTable для переданного URL, возвращает UF_URL_TO или пустую строку. Проверяет оба варианта URL (с завершающим слэшем и без) через array_unique в фильтре.
  - Метод `getCanonicalLink()` (строки 885-896): получает текущий URL через Ctx::request()->getRequestedPageDirectory(), вызывает getCanonicalLinkReplacement() для проверки замены, если замена найдена и отличается от текущего URL — использует её, иначе формирует canonical из текущего URL. Возвращает полный URL с протоколом, хостом и завершающим слэшем.

## Gotchas
- В фильтре Bitrix ORM для поиска по массиву значений используется `['=UF_URL_FROM' => array_unique([...])]` — Bitrix интерпретирует массив как условие "IN" (один из вариантов).
- Метод getCanonicalLinkReplacement() принимает URL как параметр (не получает его сам), что делает его более гибким и тестируемым.
- Проверка `$replacement != $url` в getCanonicalLink() предотвращает замену URL на самого себя (если UF_URL_TO совпадает с текущим URL).
- Канонические ссылки выводятся в TemplateHelper::setMeta() через Seo::getCanonicalLink() в теги `<link rel="canonical">` и `<meta property="og:url">`.

## Verification
- Хайлоадблок SeoCanonicalLinks создан и доступен в админке Bitrix (Настройки → Настройки продукта → Highload-блоки).
- На странице поиска, для которой есть запись в хайлоадблоке (UF_URL_FROM = текущий URL), в canonical и og:url подставляется UF_URL_TO.
- На страницах без записи в хайлоадблоке canonical формируется из текущего URL (как раньше).
- Запросы к хайлоадблоку кешируются, что не влияет на производительность.

- Мониторинг эффекта: отслеживать изменения в индексации и ранжировании страниц каталога после внедрения механизма.
