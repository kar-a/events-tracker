# Task Memory Card

Date: 2025-12-05
Task Key: 1234
Title: Правило работы с корзиной "Оплата сертификатом"

## Context
- Проблема: Необходимо реализовать автоматическое применение скидки при оплате сертификатом в корзине и обеспечить корректный учёт использованных сертификатов
- Базовое поведение: В Bitrix используется система правил работы с корзиной (OnCondSaleActionsControlBuildList) для применения скидок. Сертификаты хранятся в HL блоке `CertificatesTable` и имеют купоны, связанные с правилом скидки через `CERTIFICATE_PAYMENT_DISCOUNT_XML_ID`

## Key Decisions
- Создание кастомного правила через наследование от `CCatalogCondCtrlComplex` → позволяет интегрировать логику оплаты сертификатом в стандартную систему скидок Bitrix → пользователи могут применять сертификаты как обычные купоны
- Метод `applyToBasket` рассчитывает скидку на основе суммы сертификата и применяет через `Discount\Actions::applyToBasket` → гарантирует корректное применение скидки с учётом уже применённых скидок → пользователи видят точную сумму к оплате
- Деактивация купона и сертификата при создании заказа через событие `OnSaleOrderSaved` → предотвращает повторное использование сертификата → обеспечивает однократное использование каждого сертификата

## Code Touchpoints
- `app/local/php_interface/lib/events/sale/OnCondSaleActionsControlBuildList/CertificatePayment.php` → класс `CertificatePayment` extends `CCatalogCondCtrlComplex` → регистрирует кастомное правило "Оплата сертификатом", метод `applyToBasket` применяет скидку на сумму сертификата к корзине
- `app/local/php_interface/lib/events/inc.php` → подключение файла `CertificatePayment.php` → правило становится доступным в системе скидок Bitrix
- `app/local/php_interface/lib/app/Catalog/Certificate.php` → константа `CERTIFICATE_PAYMENT_DISCOUNT_XML_ID = 'CERTIFICATE_PAYMENT_DISCOUNT'`, метод `getCertificatePaymentDiscountRule()` → идентификация правила скидки для сертификатов
- `app/local/php_interface/lib/events/sale/Order.php` → метод `onOrderSaved` → проверяет использование правила `CERTIFICATE_PAYMENT_DISCOUNT_XML_ID` и деактивирует купоны/сертификаты при создании заказа, записывает дату использования и ID заказа в `CertificatesTable`

## Gotchas (Pitfalls)
- Метод `applyToBasket` должен учитывать уже применённые скидки через `$discountPrice` → иначе может произойти перерасчёт скидки
- При расчёте скидки используется формула `$certificateSum - $discountPrice - 10` для случаев, когда сумма корзины больше суммы сертификата → необходимо проверить логику расчёта в различных сценариях
- Деактивация сертификата происходит только при успешном создании заказа → если заказ не создан, сертификат остаётся активным
- Логирование ошибок в методе `applyToBasket` ведётся в `_certificate_payment_error` и `_applyToBasket` → нужно отслеживать эти логи для отладки
