## 1179 — Период охлаждения: фикс времени отмены

Context
- Неоплаченные заказы отменялись по базовому таймауту, без учёта удлинённого периода ожидания для рассрочки в зависимости от суммы заказа.

Key Decisions
- В кроне отмен (`app/local/cron/orders/cancel_unpaid.php`) перед отменой рассчитать увеличенный таймаут для рассрочки через `Exchange::getCreditUnpaidTimeoutMinutesByPrice($price)` и сравнить с возрастом заказа.
- Для оплат картой оставить прежний базовый порог `Exchange::MAX_MINUTES_STAY_UNPAID`.

Code Touchpoints
- `app/local/cron/orders/cancel_unpaid.php` — добавлена валидация времени с учётом суммы (paySystem ∈ `OrderHelper::CREDIT_PAY_SYSTEMS`).
- `App\Exchange::getCreditUnpaidTimeoutMinutesByPrice()` — источник правил временных порогов.

Gotchas
- Использовать `getDateInsert()->getTimestamp()` и проверять, что время создания валидно.
- Учитывать, что у заказа может быть несколько платежей; текущая логика берёт первый (`$order->getPaymentSystemId()[0]`).

Verification
- Интеграционный тест: создать заказ в рассрочку с ценой в каждом из диапазонов, убедиться, что до истечения расчётного таймаута отмена не происходит, а после — происходит.
- Smoke: для заказов по карте отмена происходит по базовому порогу.

Follow-ups
- При необходимости вынести выбор актуального платежа в helper и/или учитывать частичные оплаты.
