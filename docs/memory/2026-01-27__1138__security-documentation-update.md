## Код Touchpoints

### BeforeProlog.php

**Метод:** `checkScanners()`  
**Изменения:**
- Строки 251 и 272: добавлена фильтрация чувствительных заголовков из `$_SERVER` перед `ulogging()`
- Фильтруются: `HTTP_AUTHORIZATION`, `HTTP_COOKIE`, `HTTP_X_API_KEY`, `HTTP_X_CSRF_TOKEN`, `HTTP_X_BITRIX_CSRF_TOKEN`

**Эффект:** Предотвращение утечки чувствительных данных в логи блокировки сканеров.

### catalog.smart.filter/component.php

**Изменения:**
- Добавлен `use \App\Ctx;` в начало файла
- Строки 782, 813, 846: заменено `$_GET['bxajaxid']` на `Ctx::request()->get('bxajaxid')` с валидацией через регулярное выражение `/^[a-zA-Z0-9_-]+$/`

**Эффект:** Унификация доступа к параметрам запроса и защита от XSS через валидацию формата.

---

## Gotchas (Pitfalls)

1. **Фильтрация $_SERVER** — важно фильтровать перед каждым вызовом `ulogging()`, а не один раз, так как `$_SERVER` может изменяться между вызовами
2. **Валидация bxajaxid** — регулярное выражение должно быть строгим, чтобы предотвратить инъекции, но достаточно гибким для реальных значений Bitrix AJAX ID
3. **Удаление исправленных проблем** — после исправления нужно обновить нумерацию оставшихся проблем в документе аудита
