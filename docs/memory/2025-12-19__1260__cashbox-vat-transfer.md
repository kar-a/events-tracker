# 1260: Передача НДС в чеки

**Date**: 2025-12-19  
**Status**: Completed

## Context
При формировании чеков через кассу первого ОФД требовалось обеспечить корректную передачу НДС в фискальные данные. Для соответствия требованиям фискального законодательства и упрощения учета необходимо было установить единую ставку НДС для всех товаров в чеках.

## Key Decisions
- Жесткое присвоение НДС 20/120 для всех позиций → гарантирует единообразие НДС в чеках → соответствие требованиям ОФД и упрощение учета
- Применение к модулю кассы первого ОФД → изменения изолированы в модуле кассы → не затрагивает основную логику каталога и товаров
- Использование формулы 20/120 (НДС включен в цену) → стандартная формула для розничной торговли → соответствует практике большинства интернет-магазинов

## Code Touchpoints
- `bitrix/modules/aam.firstofdru/lib/AAM/FirstOFD.php`
  - Добавлено жесткое присвоение НДС 20/120 для всех позиций при формировании данных чека
  - Все товары в чеках теперь имеют фиксированную ставку НДС 20% включенного в цену
  - Изменение применяется независимо от настроек VAT_ID товаров в каталоге

## Gotchas (Pitfalls)
- Жесткое присвоение НДС может не соответствовать реальной ставке НДС для некоторых товаров → если в каталоге есть товары с другой ставкой НДС, это может привести к несоответствиям → необходимо убедиться, что все товары действительно имеют ставку 20%
- Изменения затрагивают только кассу первого ОФД → другие кассы могут иметь другую логику передачи НДС → при добавлении новых касс необходимо учитывать особенности каждой
- Формула 20/120 предполагает, что НДС уже включен в цену товара → если цены указаны без НДС, формула будет некорректной → необходимо убедиться, что в настройках каталога установлено VAT_INCLUDED = 'Y'

## Verification

- Убедиться, что фискальные данные корректно передаются в первый ОФД
- Проверить, что изменения не влияют на другие кассы (если они используются)
- Убедиться, что настройки каталога соответствуют формуле 20/120 (VAT_INCLUDED = 'Y')
