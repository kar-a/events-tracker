# 1185: Класс отправки SMS через Beeline

## Дата: 2025-09-30

## Контекст
На сайте используется отправка SMS через класс \App\\Sms\. Ранее использовался модуль \StreamTelecomSms\, который более не поддерживается. Требовалось реализовать новую интеграцию с Beeline API.

## Решение

### Архитектура
Реализован Service-паттерн для интеграции с Beeline:

- **\App\\Sms\\Beeline\\Service\** — HTTP-сервис для работы с API
- **\App\\Sms\\Beeline\\Beeline\** — фасад для доменной логики  
- **\App\\Sms\** — основной класс отправки SMS (namespace \App\)

### Технические детали

#### Конфигурация
\\\php

define('APP_BEELINE_SMS_LOGIN', \['BEELINE_SMS_LOGIN']);

#### API Endpoint
- **HOST:** \https://a2p-sms-https.beeline.ru/\
- **Метод:** \proto/http/rest\
- **Авторизация:** Логин/пароль в теле запроса
- **Формат:** JSON \{user, pass, action: 'post_sms', target: '+phone', message}\

### Точки вызова

1. **Авторизация** — \App\\User\\Auth\\Auth::loginOrRegister()\
   - Отправка кодов входа на сайт

2. **Уведомления заказов** — \App\\EventHandler\\Main\\OnBeforeEventSend::OrderSmsNotification()\
   - SMS при изменении статуса заказа

3. **Фоновая отправка** — \pp/local/cron/send_sms.php\
   - Обработка очереди неудачных отправок

### Логика очереди

- **\Sms::send()\** — прямая отправка, при неудаче → очередь
- **\Sms::moveStack()\** — фоновая обработка очереди
- **\SmsStackTable\** — таблица очереди с полями:
  - \UF_PHONE\ — номер телефона
  - \UF_MESSAGE\ — текст сообщения  
  - \UF_SUCCESS\ — статус отправки (Y/F)
  - \UF_COUNT_TRY\ — количество попыток
