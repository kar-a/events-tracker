Title: Установка Qdrant

## Context
- Требовалось установить Qdrant в проекте для последующей реализации поиска похожих товаров по фото.
- Qdrant — векторная база данных для similarity search; подходит для хранения эмбеддингов изображений и поиска ближайших по вектору (например, по фото пользователя).
- Окружение: dev-инфраструктура в `system/system/.dev/`, использование Docker Compose.

## Key Decisions
- Запуск Qdrant через Docker Compose в отдельной папке `system/system/.dev/qdrant/`.
- Образ: `qdrant/qdrant:latest`.
- Порты: 6333 (REST API), 6334 (gRPC), привязаны к 127.0.0.1 (доступ только с локальной машины).
- Постоянное хранилище: volume `./qdrant_storage:/qdrant/storage`, чтобы данные не терялись при перезапуске контейнера.
- Политика перезапуска: `unless-stopped`.

## Code Touchpoints
- `system/system/.dev/qdrant/compose.yaml` — единственный изменённый файл:
  - Сервис `qdrant`, контейнер `qdrant`.
  - Порты 127.0.0.1:6333:6333, 127.0.0.1:6334:6334.
  - Volume для persistence: `./qdrant_storage:/qdrant/storage`.

## Gotchas
- Доступ к Qdrant только с localhost; для доступа с других машин нужно изменить привязку портов (например, `"6333:6333"`) и учитывать безопасность.
- Папка `qdrant_storage` создаётся при первом запуске; при необходимости её можно добавить в .gitignore.
- Дальнейшие шаги: клиент к Qdrant (PHP или другой), пайплайн генерации эмбеддингов изображений товаров (модель/API), создание коллекций и индексация, API поиска похожих по фото.

## Verification
- Запуск: `docker compose -f system/system/.dev/qdrant/compose.yaml up -d`.
- Проверка: `curl http://127.0.0.1:6333/` — должен вернуть информацию о Qdrant (например, версию).
- Dashboard (если нужен): по умолчанию в образе не включён; при необходимости можно поднять отдельно или использовать REST API.
