# Task Memory Card
Date: 2026-01-26
Task Key: 1284
Title: Пропадание списка моделей при повторных открытиях мобильного меню

## Context
- При повторных открытиях мобильного меню список моделей пропадал и не отображался.
- Проблема возникала только при повторных открытиях меню (когда `mobileMenu` уже был создан), при первом открытии все работало корректно.
- Мобильное меню использует библиотеку CupertinoPane для отображения панели меню на мобильных устройствах.
- При первом открытии создается клон контейнера меню (`mobileMenuContainer`) и инициализируется CupertinoPane.
- При повторных открытиях используется уже созданный `mobileMenu` и просто показывается панель без сброса состояния.

## Key Decisions
- Добавить вызов `setStartScreen()` при повторном открытии мобильного меню для сброса состояния на начальный экран.
- Метод `setStartScreen()` уже существовал и использовался при открытии десктопного меню (`openMenu()`) и при клике на бэкдроп мобильного меню, но не вызывался при повторном открытии мобильного меню.
- Решение: добавить вызов `menu.setStartScreen()` перед показом панели при повторном открытии (в блоке `else` метода `openMobileMenu()`).

## Code Touchpoints
- `app/local/changes/template/src/js/app/AppHeaderMenu.js`:
  - Метод `openMobileMenu()` (строки 225-282):
    - При первом открытии (`if (!menu.mobileMenu)`) создается клон контейнера меню и инициализируется CupertinoPane.
    - При повторном открытии (`else`) добавлен вызов `menu.setStartScreen()` перед показом панели (строка 279).
    - Вызов `setStartScreen()` сбрасывает состояние меню: убирает активные классы у моделей и категорий, возвращает меню в начальное состояние.
  - Метод `setStartScreen()` (строки 180-192):
    - Кликает по кнопкам "Все изделия" (`category_link_all`) для сброса выбранных категорий.
    - Добавляет активный класс главным категориям (`category_item[data-type=main-category]`) для отображения начального состояния меню.
    - Использует `document.querySelectorAll` для поиска элементов, что работает как для оригинального контейнера, так и для клона (`mobileMenuContainer`).

## Gotchas
- Метод `setStartScreen()` использует `document.querySelectorAll`, который ищет элементы во всем документе, включая как оригинальный контейнер меню, так и клон (`mobileMenuContainer`). Это работает корректно, так как при вызове `setStartScreen()` активные классы убираются у всех элементов с соответствующими селекторами, включая элементы в `mobileMenuContainer`.
- При первом открытии мобильного меню `setStartScreen()` не вызывается явно, но это не проблема, так как клон создается с начальным состоянием из оригинального контейнера.
- При клике на бэкдроп мобильного меню также вызывается `setStartScreen()` (строка 272), что обеспечивает сброс состояния при закрытии меню через бэкдроп.

## Verification
- При повторных открытиях мобильного меню список моделей отображается корректно.
- Состояние меню сбрасывается на начальный экран при каждом открытии.
- Активные классы у моделей и категорий убираются при повторном открытии меню.
- Меню всегда открывается в начальном состоянии с отображением всех категорий.
- Навигация по каталогу на мобильных устройствах работает корректно.

## Follow-ups
- Проверить работу мобильного меню на различных устройствах и браузерах.
- Убедиться, что сброс состояния не влияет на производительность при частых открытиях меню.
- Рассмотреть возможность оптимизации метода `setStartScreen()` для работы только с конкретным контейнером (оригинальным или клоном) вместо поиска во всем документе, если это улучшит производительность.
