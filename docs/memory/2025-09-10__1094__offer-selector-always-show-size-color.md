# Task Memory Card

Date: 2025-09-10
Task Key: 1094
Title: Конструктор — не скрывать цвет и размеры при несовместимых комбинациях

## Context
- В карточке товара (конструктор вариантов) цвет и размер могли скрываться при недоступной текущей комбинации свойств.
- Требование: «Не скрывать цвет и размеры в конструкторе даже если нет при текущем наборе свойств» — пользователь должен видеть карту вариантов и иметь возможность перейти к доступной конфигурации.
- Базовые точки: `App\Catalog\Helper::getProductConstructor()`, шаблон `catalog.element/main/template.php`, SCSS для несовместимых значений.

## Key Decisions
- Всегда оставлять видимыми COLOR и SIZE → Предсказуемая навигация по вариантам → меньше «пропаданий» UI.
- Несовместимые значения помечать стилем (пунктир/серый), но оставлять ссылку на доступный товар → Быстрый переход к рабочей конфигурации.
- В шаблоне размеры показывать даже при `$canBuy === false` (аналогично цвету) → Консистентность поведения.
- (Опционально) Не скрывать SIZE даже при одном значении, если важно единообразие визуального ряда → Единообразный UI.

## Code Touchpoints
- Files → `app/local/php_interface/lib/app/Catalog/Helper.php` → `getProductConstructor()`
  - Логика не удаляет COLOR/SIZE при фильтрации недоступных значений.
- Files → `app/local/templates/trimiata/components/app/catalog.element/main/template.php`
  - Показ блока размеров только при `$canBuy` — требуется показать и при `!$canBuy`.
- Files → `app/local/changes/template/src/styles/template/layout/pages/product/product-prop/product-prop.scss`
  - `.product_prop_incompatible` пустой — добавить визуальное состояние «несовместимо».

## Gotchas (Pitfalls)
- Построение URL для несовместимых значений — брать первый доступный из `PROPS` (есть в helper) и не терять выбранные значения других свойств.
- Не прятать SIZE при единственном значении, если требуется единообразный UI (на усмотрение продуктовой логики).
- A11y: пометки `aria-current` для выбранного и `aria-disabled` (или визуальный эквивалент) для несовместимых значений.

## Verification
- Карточка с несколькими цветами/размерами: при выборе невозможной комбинации цвет/размер остаются видимыми; клик ведёт к доступной конфигурации.
- Карточка с недоступным текущим товаром (`$canBuy=false`): блок размеров виден, несовместимые значения помечены.
- Mobile/desktop: корректные стили, скролл, клики.

## Follow-ups
- Унифицировать отображение для категорий/моделей с одним значением SIZE.
- Добавить aria-атрибуты для доступности.
