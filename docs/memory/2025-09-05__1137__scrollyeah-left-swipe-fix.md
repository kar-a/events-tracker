## 1137: Scrollyeah — левый свайп сохраняет позицию

Context
- На мобильных при свайпе влево блок быстрых фильтров «возвращался» на старт. Вправо работал корректно.

Key Decisions
- Инерция считается в координатах `_pos` симметрично для обоих направлений; цель = `_pos + momentum`, затем кламп в `[min;max]`.
- Подавление кликов после реального драга (capture) на валу и корне.
- Очистка обработчиков клика в `destroy()`.

Code Touchpoints
- `app/local/changes/template/src/js/modules/Scrollyeah.js`
  - pointerMove: детекция драга по `|dx|>5`.
  - pointerUp: расчёт `momentum` по скорости, кламп, анимация `_animateTo`.
  - onWheel: линейное смещение `_pos` в обе стороны.
  - Click suppression + снятие обработчиков в `destroy()`.

Gotchas
- Не хранить типы событий как реальные строки в `_listeners` (используются псевдонимы `clk`, `clk_root`) — для снятия нужны явные ветки.

Verification
- Мобильный: свайп влево/вправо — позиция сохраняется; инерция плавная; клик по чипсу без драга работает.

Follow-ups
- При динамической подмене контента вызывать `Scrollyeah.forceInit(...).forEach(i => i.refresh())`.
