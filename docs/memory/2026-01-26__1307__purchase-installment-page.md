# Task Memory Card
Date: 2026-01-26
Task Key: 1307
Title: Страница покупки в рассрочку

## Context
- Страница `/purchase/` требовала полностью новой верстки в соответствии с новыми макетами дизайнера из Figma (node-id: 19154-1967, 19154-2074).
- Старая верстка не соответствовала новым макетам и не использовала единую дизайн-систему проекта через токены Figma.
- Необходимо было создать новую структуру стилей для всех блоков страницы: преимущества рассрочки, варианты рассрочки, блок "Где оформить рассрочку", FAQ.
- Блок "Где оформить рассрочку" требовал визуального выравнивания с макетом (убрать рамки, исправить цвета, добавить ссылки на магазины).

## Key Decisions
- Выполнить полностью новую верстку всей страницы `/purchase/` в соответствии с макетами Figma.
- Использовать токены Figma для всех цветов (`@include _color('primary/tints/600')`, `@include _color('black/tints/800')`) и типографики (`@include _style('text-l/font-normal')`, `@include _style('headers/small/uppercase/h2')`) для обеспечения соответствия дизайн-системе.
- Создать новые стили для всех блоков страницы с полной адаптивностью через медиа-запросы.
- Для блока "Где оформить рассрочку": перевести контент на PHP массив `$where_items` по аналогии с другими блоками (`$options`, `$faq`), использовать CSS Grid layout вместо row-based структуры, убрать рамки карточек согласно макету, добавить кликабельные ссылки на страницы контактов для адресов магазинов.
- Использовать Figma MCP для получения точных спецификаций дизайна и токенов.

## Code Touchpoints
- `app/purchase/index.php`: 
  - Структура страницы сохранена, но контент блока "Где оформить рассрочку" переведен на PHP массив `$where_items` с данными всех элементов (4 элемента: 2 магазина, оформление по телефону, оформление на сайте).
  - Каждый элемент массива содержит: `title`, `title_class`, `info`, `additional_info`, `url`, `card_class`.
  - Для элементов с `url` заголовок выводится как ссылка `<a>`, для остальных — как `<div>`.
  - Структура HTML изменена с `purchase_where__row` на `purchase_where__grid`.
  - Вывод через двойной `foreach` заменен на одинарный `foreach` по массиву элементов.
- `app/local/changes/template/src/styles/template/layout/pages/purchase/purchase.scss`:
  - Создана полностью новая структура стилей для всех блоков страницы в соответствии с макетами Figma.
  - **Контейнеры блоков**: `.purchase_benefits_container`, `.purchase_options_container`, `.purchase_where_container`, `.purchase_faq_container` — flex-контейнеры с центрированием, gap 3.125rem (1.5rem на мобильных).
  - **Блок преимуществ**: `.purchase_benefits__items` (flex column, gap 2.5rem), `.purchase_benefits__row` (flex, gap 4.5rem), `.purchase_benefits__item` (flex, gap 1.25rem), `.purchase_benefits__icon` (4.5rem × 4.5rem, цвет `primary/tints/600`), `.purchase_benefits__text` (типографика `text-l/font-normal`, цвет `black/tints/800`).
  - **Блок вариантов рассрочки**: `.purchase_options_wrapp` (flex, gap 4.5rem, адаптивность на мобильных), `.purchase_options__card` (flex column, gap 0.75rem), `.purchase_options__card_name` (типографика `text-xl/font-medium`), `.purchase_options__card_list` (flex column, gap 0.25rem).
  - **Блок "Где оформить рассрочку"**: `.purchase_where__grid` (CSS Grid, 2 колонки, gap 3.125rem × 1.5rem, 1 колонка на мобильных), `.purchase_where__card` (flex column, gap 0.75rem, без рамок согласно макету), `.purchase_where__card_title` (типографика `headers/small/uppercase/h2`, uppercase), `.purchase_where__card_info` (типографика `text-m/font-medium`, цвет `black/tints/800`), `.purchase_where__card_link` и `.purchase_where__card_phone` (hover-эффекты с изменением цвета на `primary/tints/600`).
  - Все цвета и типографика используют токены Figma через миксины `@include _color()` и `@include _style()`.
  - Реализована полная адаптивность через медиа-запросы `@include media-breakpoint-down(lg/md/sm)`.

## Verification
- Страница `/purchase/` визуально соответствует новым макетам Figma (node-id: 19154-1967, 19154-2074).
- Все блоки используют токены Figma для цветов и типографики, обеспечивая соответствие дизайн-системе.
- Блок "Где оформить рассрочку" отображается без рамок, с правильными цветами заголовков (золотой для адресов магазинов), использует Grid layout.
- Адреса магазинов являются кликабельными ссылками, ведущими на соответствующие страницы контактов.
- На мобильных устройствах все блоки корректно адаптируются (Grid переходит в 1 колонку, flex-элементы меняют направление).
- Hover-эффекты работают для ссылок и телефонных номеров (изменение цвета на `primary/tints/600`).
- Контент блока "Где оформить рассрочку" управляется через PHP массив, изменения можно вносить без правки HTML.
- Все стили используют единицы `rem` и токены Figma, что обеспечивает консистентность с остальным сайтом.

## Follow-ups

- Рассмотреть возможность вынесения данных в отдельный конфигурационный файл или базу данных для управления через админ-панель.
- При обновлении токенов Figma через `npm run figma` все стили страницы автоматически синхронизируются с новой дизайн-системой.
