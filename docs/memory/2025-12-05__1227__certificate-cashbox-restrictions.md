# Task Memory Card

Date: 2025-12-05
Task Key: 1227
Title: Ограничение онлайн касс, тестовые сертификаты, проверка чеков

## Context
- Задача: реализовать ограничение онлайн касс для заказов с подарочными сертификатами, добавить поддержку тестовых сертификатов и проверку чеков при печати
- Проблема: онлайн кассы должны работать только для обычных заказов, а не для сертификатов (сертификаты — виртуальный товар, не требует фискальных документов)
- Ограничение: необходимо различать заказы сертификатов от обычных заказов и применять ограничения касс только к нужным типам заказов

## Key Decisions
- Создан класс ограничения `CertificateOrder` для касс → позволяет настраивать кассы с ограничением "только для сертификатов" или "исключить сертификаты"
- Добавлена константа `APP_CERTIFICATES_TEST` для тестовых сертификатов → позволяет обходить валидацию суммы/шага при тестировании
- Метод `OrderHelper::isCertificateOrder()` определяет заказы сертификатов по XML_ID товара в корзине → единая точка проверки типа заказа
- Событие `onSaleCheckPrinted` обрабатывает печать чеков для сертификатов → логирует данные чека и фискальные данные для отправки в 1C (пока закомментировано)
- Автоматическое изменение статуса заказа сертификата при создании → устанавливается статус `STATUS_WAIT_PAYMENT_CERTIFICATE` вместо стандартного

## Gotchas (Pitfalls)
- Ограничение касс работает только если касса настроена с этим ограничением в админке Bitrix → необходимо вручную настроить кассы
- Метод `isCertificateOrder()` принимает как объект Order, так и ID заказа → автоматически загружает заказ при необходимости
- Проверка чеков в `onCheckPrinted` пока не отправляет данные в 1C (закомментирован `Exchange::addToStackReceipt`) → требуется доработка интеграции
- Тестовые сертификаты обходят валидацию суммы, но не влияют на другие проверки → необходимо контролировать использование константы
- Статус заказа сертификата меняется только при создании нового заказа (`IS_NEW`) → существующие заказы не затрагиваются
