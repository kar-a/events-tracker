# Task Memory Card

Date: 2025-11-23 (обновлено: 2026-01-09)
Task Key: 1217
Title: Рефакторинг js заказа

## Context
- Проблема: Процедурный код в `app/local/templates/trimiata/js/checkout.js` не соответствовал стилю проекта (классовая структура), использовал смешанные селекторы (классы, data-атрибуты), был сложен для поддержки
- Проблема (расширение): Все JS чанки загружались на каждой странице, даже если не нужны (например, `datepicker.js` на странице `/company/`), что увеличивало время загрузки и расход трафика
- Ограничения: Необходимо сохранить всю функциональность (переключение способов доставки/оплаты, обновление информации, оформление заказа, обработка подарков, скидок)
- Базовое поведение: Форма заказа в `app/local/components/koptelnya/checkout/templates/.default/form.php`, AJAX обработка в `app/local/components/koptelnya/checkout/ajax.php`

## Key Decisions
- Решение: Создать класс `AppOrder` по аналогии с другими классами проекта (`AppAuthorization`, `AppBasket`, `AppForm`)
  - Рациональное: Соответствие стилю проекта, улучшение читаемости и поддерживаемости
  - Влияние: Единообразный подход к организации JS кода, упрощение тестирования и отладки
- Решение: Перевести все селекторы на `data-role` атрибуты
  - Рациональное: Унификация селекторов, независимость от CSS классов, упрощение рефакторинга
  - Влияние: Обновлены шаблон формы и AJAX ответы для генерации HTML с data-role
- Решение: Упростить код, убрав одноразовые переменные, добавив метод `setLoading`
  - Рациональное: Улучшение читаемости, уменьшение дублирования кода
  - Влияние: Более компактный и понятный код
- Решение: Исправить отправку формы через AJAX, убрав `preventDefault` из `AppForm`
  - Рациональное: Позволить валидатору jQuery корректно обрабатывать submit события
  - Влияние: Форма корректно отправляется через AJAX без перехода на страницу
- Решение (расширение): Реализовать паттерн lazy loading через `_lazy` factory в `Router.js`
  - Рациональное: `import()` в объекте `_modules`/`_assets` выполнялся сразу при загрузке `Router.js`, что приводило к загрузке всех чанков на каждой странице. Паттерн `_lazy` создает Promise только при первом вызове функции-обертки
  - Влияние: JS модули загружаются только когда они действительно нужны (при вызове `Router.modules.X()` или `Router.assets.X()`), что значительно уменьшает размер начальной загрузки
- Решение (расширение): Отключить автоматические механизмы Webpack для ручного управления чанками
  - Рациональное: `AutomaticPrefetchPlugin` и `webpackPrefetch: true` в динамических импортах приводили к префетчу всех чанков. Отключение `defaultVendors` и `default` cache groups позволяет полностью контролировать имена чанков через `webpackChunkName`
  - Влияние: Предсказуемые имена чанков (например, `js/catalog.js`, `js/assets/forms.js`), отсутствие безымянных чанков типа `255.js`, отсутствие нежелательного префетча
- Решение (расширение): Группировка модулей и ассетов в логические чанки
  - Рациональное: Модули, которые обычно используются вместе (например, каталог: `AppBasket`, `AppCatalogSorter`, `AppSmartFilter`, `AppWishlist`), объединены в один чанк `js/catalog`
  - Влияние: Уменьшение количества HTTP-запросов, улучшение кеширования, более эффективная загрузка связанных модулей

## Gotchas (Pitfalls)
- Важно: `id="checkout-form"` должен быть на контейнере `[data-role="order-checkout"]`, а не на форме. `AppForm` ищет контейнер по ID, затем форму внутри него
- Важно: Селектор формы в `AppOrder.elements.form` должен быть `'form'`, а не `'#checkout-form'`, так как форма находится внутри контейнера
- Важно: Не использовать `preventDefault` в `AppForm` до инициализации валидатора jQuery, иначе валидатор не сможет обработать submit событие
- Важно: При переопределении метода `_send` в `AppOrder` необходимо возвращать Promise для корректной обработки цепочки промисов
- Важно: Все AJAX ответы должны генерировать HTML с `data-role` атрибутами для соответствия селекторам в `AppOrder`
- Важно (lazy loading): В `Router.js` модули и ассетсы должны быть обернуты в `_lazy()`, иначе `import()` выполнится сразу при загрузке `Router.js`
- Важно (lazy loading): При использовании Router необходимо вызывать функции: `Router.modules.X()` вместо `Router.modules.X.then(...)`, так как `Router.modules.X` теперь функция, а не Promise
- Важно (webpack): Использовать `dist/js/nice-select2.js` вместо `src/js/nice-select2.js` для NiceSelect, так как `src/js` импортирует SCSS и порождает отдельный CSS-чанк (`forms.min.css`)
- Важно (webpack): Все динамические импорты должны иметь явный `webpackChunkName` для предсказуемых имен чанков
