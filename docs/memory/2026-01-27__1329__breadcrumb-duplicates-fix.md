# Задача 1329: Дублирование хлебных крошек поиска

**Дата:** 2026-01-27  
**Задача:** 1329  
**Тип:** Исправление ошибки, UI/UX

---

## Контекст

**Проблема:** На странице поиска происходило дублирование элементов хлебных крошек. Элементы "Каталог" и "Поиск" выводились дважды, что создавало визуальную путаницу и некорректную структуру навигации.

**Пример дублирования:**
```html
<div class="breadcrumb__item">Каталог</div>
<div class="breadcrumb__item">Поиск</div>
<div class="breadcrumb__item">Каталог</div> <!-- дубликат -->
<div class="breadcrumb__item">Поиск</div> <!-- дубликат -->
```

**Ограничения:** 
- Необходимо сохранить существующую логику формирования хлебных крошек
- Проверка должна работать для всех страниц, не только для поиска
- Не должно влиять на производительность

**Базовое поведение:** 
- Компонент `bitrix:breadcrumb` использует массив `$arResult` с элементами хлебных крошек
- Элементы добавляются через `$APPLICATION->AddChainItem()` в различных компонентах
- Шаблон `breadcrumb/main/template.php` выводит все элементы из `$arResult`

---

## Ключевые решения

### Проверка дубликатов по комбинации ссылки и названия

**Решение:** Использовать один массив `$filteredItems` с ключами-хешами (MD5 от комбинации LINK и TITLE) для быстрой проверки дубликатов через `isset()`.

**Обоснование:** 
- Комбинация ссылки и названия уникально идентифицирует элемент хлебных крошек
- MD5-хеш обеспечивает быструю проверку через ключи массива (O(1))
- Один массив вместо двух упрощает код и снижает потребление памяти
- `array_values()` преобразует ассоциативный массив в числовой для корректного цикла

**Влияние:** Устраняет дубликаты без влияния на производительность, упрощает код.

## Code Touchpoints

### `app/local/templates/trimiata/components/bitrix/breadcrumb/main/template.php`

**Секция:** Фильтрация дубликатов (строки 24-31)
- Создается массив `$filteredItems` для хранения уникальных элементов
- Для каждого элемента из `$arResult` вычисляется MD5-хеш от комбинации `LINK` и `TITLE`
- Проверка дубликатов через `isset($filteredItems[$itemKey])`
- Элементы сохраняются с ключом-хешем: `$filteredItems[$itemKey] = $arItem`
- После фильтрации массив преобразуется в числовые индексы: `$filteredItems = array_values($filteredItems)`

**Секция:** Вывод элементов (строки 33-78)
- Цикл по отфильтрованному массиву `$filteredItems`
- Определение последнего элемента через `$isLast = ($key === $filteredCount - 1)`
- Логика вывода осталась без изменений

## Gotchas (Pitfalls)

1. **Использование `??` для LINK**: Необходимо использовать `($arItem['LINK'] ?? '')` для корректной обработки элементов без ссылки (последний элемент).

2. **MD5-хеш для ключа**: Использование MD5 обеспечивает уникальность ключа и быструю проверку, но важно использовать одинаковую комбинацию полей (LINK + TITLE) для корректного определения дубликатов.

3. **`array_values()` после фильтрации**: Необходимо преобразовать ассоциативный массив в числовой для корректной работы цикла `foreach` с определением последнего элемента через `$key === $filteredCount - 1`.

4. **Проверка `isset()` вместо `array_key_exists()`**: Использование `isset()` быстрее и достаточно для проверки наличия ключа в массиве.

5. **Сохранение порядка элементов**: `array_values()` сохраняет порядок элементов, что важно для корректного отображения хлебных крошек.
