# Task Memory Card

Date: 2026-01-10
Task Key: 1280
Title: Фильтрация по цене при вводе одного значения вручную

## Context
При вводе только одного значения цены в фильтре (например, только максимальное значение "до") в другом поле появлялось значение NaN, и фильтрация не происходила. Проблема возникала при ручном вводе значения в одно из полей цены без заполнения второго поля. Также при стирании одного поля, если второе было заполнено, не происходила автоматическая подстановка минимального или максимального значения.

## Problem
1. В обработчике события `change` для поля "до" (`inputTo`) при пустом поле "от" код пытался установить значение в поле "от", но `parseInt('')` возвращает `NaN`, и это значение устанавливалось в поле. Аналогичная проблема возникала и в обработчике для поля "от" при пустом поле "до".
2. Коллбэки слайдера (`onChange` и `onFinish`) перезаписывали пустые поля при программном обновлении слайдера через `ionRangeSlider('update')`, что приводило к появлению некорректных значений.
3. При стирании одного поля, если второе было заполнено, не происходила автоматическая подстановка min/max значения.

## Solution
Исправлена логика обработчиков `change` для полей цены (`inputFrom` и `inputTo`):
- Добавлена проверка на пустое значение перед парсингом: если поле пустое, значение не парсится и не устанавливается в другое поле
- Для слайдера используется значение по умолчанию (min/max), но значение не устанавливается в пустое поле ввода
- Добавлена проверка на `isNaN()` после парсинга для дополнительной защиты
- Добавлен флаг `_updatingSlider` для предотвращения перезаписи пустых полей коллбэками слайдера при программном обновлении
- При стирании одного поля, если второе заполнено, автоматически подставляется min (для "от") или max (для "до")

Изменения:
- В обработчике `inputTo.addEventListener('change')`: если поле "от" пустое, значение не устанавливается в него, используется `min` только для слайдера. При стирании `inputTo`, если `inputFrom` заполнен, подставляется `max`.
- В обработчике `inputFrom.addEventListener('change')`: аналогичная логика для поля "до". При стирании `inputFrom`, если `inputTo` заполнен, подставляется `min`.
- Добавлен флаг `_updatingSlider`, который устанавливается перед обновлением слайдера и сбрасывается через `setTimeout`, чтобы коллбэки не перезаписывали пустые поля.

## Key Decisions
- Не устанавливать значение в пустое поле ввода — если пользователь не ввел значение, поле должно оставаться пустым (кроме случая, когда второе поле заполнено)
- Использовать значения по умолчанию (min/max) только для слайдера — это обеспечивает корректную работу слайдера при частичном заполнении полей
- Проверка на `isNaN()` после парсинга — дополнительная защита от некорректных значений
- Флаг `_updatingSlider` для предотвращения конфликтов между программным обновлением слайдера и его коллбэками
- Автоматическая подстановка min/max при стирании одного поля, если второе заполнено — улучшает UX и предотвращает необходимость ручного ввода граничных значений

## Code Touchpoints
- `app/local/changes/template/src/js/app/AppSmartFilter.js` — метод `initSliders()`: исправлены обработчики `change` для полей `inputFrom` и `inputTo`, добавлен флаг `_updatingSlider` в коллбэки `onChange` и `onFinish` слайдера (строки 417-558)

## Effects
- При вводе только одного значения цены (минимального или максимального) в другом поле больше не появляется NaN
- Фильтрация по цене работает корректно при частичном заполнении полей
- При стирании одного поля, если второе заполнено, автоматически подставляется граничное значение (min/max)
- Коллбэки слайдера не перезаписывают пустые поля при программном обновлении
- Улучшен пользовательский опыт работы с фильтром каталога

## Intent
Исправить проблему с фильтрацией по цене при ручном вводе одного значения или стирании одного из полей. Теперь пользователи могут вводить только одно значение цены без появления ошибок, а при стирании одного поля автоматически подставляется граничное значение, что улучшает пользовательский опыт работы с фильтром каталога.
