## 1262 — Убрать автоподстановку цены в фильтр

Date: 2026-01-04
Task Key: 1262
Title: Убрать автоподстановку цены в фильтр

## Context
- При загрузке страницы каталога поля ввода "От" и "До" в фильтре цены автоматически заполнялись значениями из слайдера через обработчик `onStart` в конфигурации ionRangeSlider
- Пользователи хотели видеть все товары без предустановленных границ цены и устанавливать диапазон вручную
- Слайдер цены находится в компоненте `app:catalog.smart.filter`, инициализация через `AppSmartFilter.initSliders()`

## Key Decisions
- Удален обработчик `onStart` из конфигурации слайдера цен → Поля остаются пустыми при загрузке → Пользователь получает полный контроль над диапазоном цен
- Оставлены обработчики `onChange` и `onFinish` → Поля заполняются только при взаимодействии пользователя → UX остается интуитивным

## Code Touchpoints
- `app/local/templates/trimiata/js/AppSmartFilter.js::initSliders()` — удален блок `onStart` из объекта `opts` (строки ~415-422 в старой версии)

## Gotchas (Pitfalls)
- Поля ввода цены могут остаться пустыми, если пользователь не взаимодействует со слайдером — это ожидаемое поведение
- При AJAX-обновлении фильтра поля также остаются пустыми, если не было взаимодействия

## Verification
- Открыть страницу каталога с фильтром цены
- Проверить, что поля "От" и "До" пустые при загрузке
- Переместить слайдер — поля должны заполниться
- Ввести значения вручную — слайдер должен обновиться
- Применить фильтр и проверить, что товары фильтруются корректно
