## 1094c: Конструктор — убрана пунктирная обводка у недоступных цветов

Context
- Продолжение задачи 1094 («не скрывать цвет и размеры при несовместимых комбо») и подзадачи 1094b (всегда показывать только цвет). Требование: убрать пунктирную обводку у недоступных значений цвета, сохранив видимость всех вариантов и кликабельность для перехода к доступной конфигурации.

Key Decisions
- Недоступные значения цвета не маркируем пунктиром/спец‑классом: визуальный шум снижен, смысловое различие остаётся через активное выделение выбранного цвета.
- Сохраняем ссылку `href` на URL варианта для быстрого перехода к доступной конфигурации.

Code Touchpoints
- `app/local/templates/trimiata/components/app/catalog.element/main/template.php`
  - В блоке цвета (case 'COLOR') удалены несовместимые классы:
    - убрано добавление `product__color_circle_incompatible`/`circle_incompatible` у недоступных значений;
    - оставлены базовые классы `product__color_circle` и `circle`; активный — `product__color_circle_active`/`circle_active`.

Gotchas
- Инвариант 1094: не скрывать значения, даже если текущая комбо недоступна — сохранить.
- Не менять логику размеров/прочих свойств в рамках этой итерации.

Verification
- Детальная страница: при недоступности части цветов — все значения видны; у недоступных отсутствует пунктир/«инкомпатибл» обводка; активный цвет остаётся выделенным.
- Клик по недоступному цвету приводит к доступной конфигурации (корректный URL). Регрессии не затрагивают размеры/вес.
- Поведение идентично в quick buy (COMPACT_MODE) и полной карточке, если используется общий шаблон секции цвета.

Follow-ups
- При необходимости уточнить гайд по визуальной маркировке недоступных значений (серый тон/подсказка) без пунктира.
