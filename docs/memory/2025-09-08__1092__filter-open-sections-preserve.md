Date: 2025-09-08
Task: 1092 — Не закрывать открытые пункты фильтра при обновлении контента

Context
- В каталоге при изменении чекбоксов фильтра блок смарт‑фильтра перерисовывается по AJAX.
- Открытые ранее секции (Bootstrap collapse) закрывались, терялся контекст пользователя.
- В задаче 1097 уже сохраняем прокрутку групп чекбоксов; теперь добавляем сохранение «открытости» секций.

Key Decisions
- Перед заменой HTML фильтра сохранить Set id всех открытых `.collapse.show`.
- После реинициализации фильтра восстановить: добавить `show` на контент, снять `collapsed` с заголовка‑тогглера, поставить `aria-expanded="true"`.
- Логику восстановления разместить в `initElements()`, чтобы работала при каждой реинициализации.

Code Touchpoints
- `app/local/templates/trimiata/js/AppSmartFilter.js`
  - В блоке обработки ответа `smartfilter`: сбор `openedIds` → `component._openedIds` перед заменой HTML; далее обычная переинициализация и восстановление прокрутки (из 1097).
  - В `initElements()`: если есть `this._openedIds`, восстановить состояние collapse (контент + тогглер).

Gotchas
- Если id секции изменится между обновлениями (редкий случай) — восстановление не сработает для этой секции, но не ломает поведение.
- Не трогаем серверную логику `DISPLAY_EXPANDED`; открытое состояние — только клиентская «память».

Verification
- Открыть 2–3 секции, выбрать чекбокс в другой секции → после обновления все ранее открытые секции остаются открытыми.
- Прокрутка внутри групп чекбоксов сохраняется (инвариант 1097).
- Десктоп/мобильный режим (панель) — одинаково корректно.

Follow-ups
- При желании можно сохранять `openedIds` в `sessionStorage` для восстановления между навигациями (не требуется по ТЗ).
