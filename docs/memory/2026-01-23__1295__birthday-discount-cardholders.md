# Task Memory Card
Date: 2026-01-23
Task Key: 1295
Title: Скидка на день рождения для владельцев карт

## Context
- Необходимо создать правило работы с корзиной для автоматического применения скидки на день рождения владельцам карт лояльности.
- Правило должно применяться только для пользователей с указанным днем рождения в профиле (`PERSONAL_BIRTHDAY`) и активной дисконтной картой.
- Период действия скидки: 2 дня до дня рождения и 3 дня после (включительно).
- Величина скидки должна настраиваться в процентах в настройках правила работы с корзиной в админке Bitrix.
- Правило должно работать как для авторизованных пользователей сайта, так и для внешних пользователей через ImShop API.

## Key Decisions
- Использовать событие `OnCondSaleActionsControlBuildList` для регистрации кастомного правила работы с корзиной.
- Создать класс `BirthdayDiscount`, наследующий `CCatalogCondCtrlComplex` для интеграции с системой правил Bitrix.
- Величину скидки сделать настраиваемой через поле ввода в админке (процент), аналогично `DiscountLimit.php`.
- Количество дней до и после дня рождения сделать константами класса (`DAYS_BEFORE = 2`, `DAYS_AFTER = 3`).
- Использовать `UserHelper::getDiscountCard()` для проверки наличия активной карты лояльности.
- Использовать `UserTable::getRow()` для получения `PERSONAL_BIRTHDAY`, который возвращает объект `Date`.
- Метод `isBirthdayPeriod()` проверяет период в текущем году, а если день рождения уже прошел — в следующем году.
- Поддержка внешних пользователей через метод `getExternalUserId()`, который извлекает `externalUserId` из запросов ImShop (`create-order`, `calculate-basket`).
- Применение скидки через `Discount\Actions::applyToBasket()` с типом `VALUE_TYPE_PERCENT`.

## Code Touchpoints
- `app/local/php_interface/lib/events/sale/OnCondSaleActionsControlBuildList/BirthdayDiscount.php`:
  - Класс `BirthdayDiscount` расширяет `CCatalogCondCtrlComplex`.
  - Константы: `DAYS_BEFORE = 2`, `DAYS_AFTER = 3`, `DISCOUNT_PERCENT_DEFAULT = 10`.
  - Метод `GetControlShow()`: определяет отображение правила в админке с полем ввода процента скидки.
  - Метод `GetControls()`: определяет структуру правила с `PREFIX = 'Скидка на день рождения %'`.
  - Метод `Generate()`: генерирует PHP-код для проверки условия, вызывает `applyToBasket()` с процентом скидки из настроек.
  - Метод `applyToBasket()`:
    - Получает ID пользователя через `getExternalUserId()` или из глобального `$USER`.
    - Проверяет наличие карты через `UserHelper::getDiscountCard()`.
    - Получает день рождения через `UserTable::getRow(['select' => ['PERSONAL_BIRTHDAY']])`.
    - Проверяет период через `isBirthdayPeriod()`.
    - Применяет скидку к каждому товару через `Discount\Actions::applyToBasket()`.
  - Метод `isBirthdayPeriod(Date $birthdayDate)`:
    - Вычисляет timestamp дня рождения в текущем году.
    - Проверяет, попадает ли текущая дата в период (день рождения ± дни).
    - Если день рождения уже прошел в текущем году, проверяет период в следующем году.
    - Использует `24 * 60 * 60` для вычисления секунд в сутках (понятное выражение вместо магического числа 86400).
  - Метод `getExternalUserId()`: извлекает `externalUserId` из JSON-запроса для методов `create-order` и `calculate-basket`.
- `app/local/php_interface/lib/events/inc.php`:
  - Добавлен `require('sale/OnCondSaleActionsControlBuildList/BirthdayDiscount.php');` для загрузки класса.

## Gotchas
- `PERSONAL_BIRTHDAY` из `UserTable::getRow()` уже является объектом `Bitrix\Main\Type\Date`, не требует дополнительного парсинга.
- Период скидки рассчитывается относительно дня рождения в текущем году, но если день рождения уже прошел, проверяется период в следующем году (для случаев в конце года).
- Метод `isBirthdayPeriod()` должен быть публичным (`public static`), так как вызывается из сгенерированного кода Bitrix.
- Использование `use` вместо FQCN для `UserTable` и других классов для упрощения кода.
- Убраны одноразовые переменные для упрощения и читаемости кода.

## Verification
- Правило "Скидка на день рождения" отображается в списке правил работы с корзиной в админке Bitrix.
- В настройках правила можно указать процент скидки (числовое значение).
- Скидка применяется только для пользователей с указанным днем рождения и активной картой лояльности.
- Скидка применяется в период 2 дня до и 3 дня после дня рождения.
- Скидка работает для авторизованных пользователей на сайте.
- Скидка работает для внешних пользователей через ImShop API (методы `create-order` и `calculate-basket`).
- Скидка применяется к каждому товару в корзине в указанном проценте.
