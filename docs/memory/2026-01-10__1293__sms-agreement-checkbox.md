# Task Memory Card

Date: 2026-01-10
Task Key: 1293
Title: Согласие на получение СМС

## Context
В окне авторизации необходимо было добавить обязательный чекбокс "Я даю согласие на получение СМС-сообщений" для соответствия требованиям законодательства. Чекбокс должен быть обязательным только при авторизации по телефону, но не должен блокировать переключение между формами авторизации (телефон ↔ email). Значение чекбокса должно сохраняться в пользовательское свойство `UF_SMS_AGREEMENT`.

## Problem
1. При добавлении обязательного чекбокса с валидацией через jQuery Validate возникала проблема: при переключении между формами авторизации (телефон ↔ email) валидация чекбокса срабатывала и блокировала переключение, требуя обязательного заполнения чекбокса даже при переходе на форму email-авторизации, где чекбокс не нужен.
2. Код для отключения валидации чекбокса при переключении был продублирован в двух местах (обработчики переключения на email и обратно на телефон), что усложняло поддержку.
3. Отсутствовала константа для кода свойства пользователя, что могло привести к ошибкам при использовании строкового литерала в разных местах кода.

## Solution
1. Добавлен обязательный чекбокс "Я даю согласие на получение СМС-сообщений" в форму авторизации по телефону (`login_or_register.php`).
2. Настроена валидация чекбокса через jQuery Validate с правилом `required: true` и кастомным сообщением об ошибке.
3. Создана функция `disableSmsAgreementValidation()` в классе `AppAuthorization` для отключения обязательности чекбокса и очистки ошибок валидации при переключении между формами.
4. Добавлена константа `UserHelper::SMS_AGREEMENT_PROP_CODE = 'UF_SMS_AGREEMENT'` для единообразного использования кода свойства.
5. Обновлена логика сохранения значения чекбокса в `Auth::loginOrRegister()` для сохранения значения при регистрации и обновлении существующего пользователя.
6. Созданы скрипты миграции:
   - `create_sms_agreement_field.php` — создание пользовательского свойства `UF_SMS_AGREEMENT`
   - `migrate_sms_newsletter_to_agreement.php` — миграция данных из `UF_SMS_NEWSLETTER_AGREEMENT` в `UF_SMS_AGREEMENT`
7. Обновлена форма редактирования профиля в личном кабинете для использования нового свойства вместо `UF_SMS_NEWSLETTER_AGREEMENT`.

## Key Decisions
- Чекбокс обязателен только при авторизации по телефону — при переключении на email-авторизацию валидация отключается, чтобы не блокировать переключение
- Использование jQuery Validate для валидации — стандартный подход в проекте, обеспечивает единообразное поведение с другими полями формы
- Вынесение логики отключения валидации в отдельную функцию — устранение дублирования кода, упрощение поддержки
- Использование константы для кода свойства — предотвращение ошибок при использовании строкового литерала, единообразие кода
- Миграция данных из старого свойства — сохранение существующих данных пользователей при переходе на новое свойство
