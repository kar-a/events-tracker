# 1351: Логирование отправки всех СМС, источник отправки

## Context
- Нужно логировать отправку СМС так же, как отправку заказов: запись всегда попадает в очередь, затем отправляется воркером, что даёт полный лог.
- Дополнительно требуется фиксировать источник СМС: `сайт` или `мобильное приложение` (imshop) — всегда.

## Key Decisions
- `Sms::send()` перестал отправлять СМС напрямую: теперь он **всегда** пишет запись в HL `SmsStack` (полный лог) и возвращает статус постановки в очередь.
- Отправка выполняется только через `Sms::moveStack()` (cron `send_sms.php`): запись помечается как `UF_SUCCESS=Y/F`, считаются попытки (`UF_COUNT_TRY`).
- Источник добавлен как обязательное поле `UF_SOURCE` (значения: `сайт`, `мобильное приложение`), участвует в уникальности записи.

## Code Touchpoints
- `app/local/php_interface/lib/app/Sms/Sms.php` → очередь+логирование, `SOURCE_*`, корректная отправка из воркера.
- `app/local/php_interface/lib/app/Sms/Beeline/Beeline.php` → убран отладочный вывод, фасад провайдера.
- `app/local/php_interface/lib/app/User/Auth/Auth.php` → авторизация по телефону/почте передаёт источник `сайт`.
- `app/local/php_interface/lib/events/main/OnBeforeEventSend.php` → СМС по заказам/сертификатам передают источник `сайт`.
- `app/local/changes/db/1351/add_sms_source_field.php` → добавление UF поля `UF_SOURCE` в HL `SmsStack` (idempotent).
- `app/local/cron/send_sms.php` → воркер отправки (`Sms::moveStack()`).

## Gotchas (Pitfalls)
- Для динамических сценариев важно не полагаться на «успех прямой отправки»: все попытки должны оставаться в HL‑логе.
- Уникальность теперь учитывает источник: одинаковые `phone+message` из разных источников не должны схлопываться в одну запись.

## Verification
- Вызвать `Sms::send(...)` → убедиться, что создаётся запись в HL `SmsStack` с `UF_SOURCE`.
- Запустить `app/local/cron/send_sms.php` → запись должна перейти в `UF_SUCCESS=Y` или `F`, а при ошибке увеличиться `UF_COUNT_TRY`.

## Follow-ups
- Для вызовов из imshop при появлении соответствующего сценария передавать `Sms::SOURCE_IMSHOP`.
