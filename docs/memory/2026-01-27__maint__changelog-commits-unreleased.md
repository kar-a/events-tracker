# Task Memory Card

Date: 2026-01-27
Task Key: maint-changelog-commits
Title: Проставление хешей коммитов для задач в разделе «Не в релизе»

## Context
- В `docs/CHANGELOG.md` для задач в разделе «Не в релизе» были описаны Files/Effects/Intent, но часто отсутствовали ссылки на конкретные коммиты (`commits: ...`).
- Это усложняло трассировку изменений от задач к git-истории и подготовку релизных заметок.
- Текущая задача — дополнить Intent всех релевантных записей в «Не в релизе» короткими SHA коммитов из `remotes/trimiata/main`.

## Key Decisions
- Использовать поиск по git-log с фильтром по номеру задачи в сообщении коммита → это минимально-инвазивный и воспроизводимый способ найти связанные изменения.
- Добавлять хеши в конец строки Intent в формате `(commits: \`abc123456\`, \`def789012\`)` → единообразный формат, уже применяемый в CHANGELOG.
- Обновлять только раздел «Не в релизе», не трогая уже зафиксированные релизы → снижает риск конфликтов и путаницы в исторических записях.

## Code Touchpoints
- `docs/CHANGELOG.md` → для каждой задачи в блоках Security/Fixed/Changed/Added в разделе «Не в релизе` дополнен Intent списком коротких SHA коммитов, найденных в `remotes/trimiata/main`.
- `.github/commit-summary.txt` → перезаписывается под конкретный коммит, опираясь на актуальные Effects/Intent из CHANGELOG.

## Gotchas (Pitfalls)
- Не все коммиты по задаче могут иметь одинаковый формат сообщения; предпочтительно использовать те, где явно указан номер задачи.
- Важно не добавлять хеши из веток, которые ещё не слиты в `main`, чтобы не ссылаться на нестабильные изменения.
- При последующих правках задач в «Не в релизе» нужно не забывать дополнять или обновлять список хешей в Intent.

## Verification
- Визуально просмотреть раздел «Не в релизе» в `docs/CHANGELOG.md` и убедиться, что у каждой записи есть `(commits: ...)` в строке Intent.
- При необходимости проверить несколько примеров: взять указанный SHA, выполнить `git show <hash>` и убедиться, что он действительно относится к описанной задаче.

## Follow-ups
- При добавлении новых задач в раздел «Не в релизе» сразу заполнять поле Intent с хешами, не откладывая на отдельную техзадачу.
- Автоматизировать проверку наличия `commits:` в Intent через линтер/CI, чтобы предупреждать о забытых записях.
