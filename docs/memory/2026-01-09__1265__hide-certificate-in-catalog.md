# Task Memory Card
Date: 2026-01-09
Task Key: 1265
Title: Скрыть сертификат в списке товаров

## Context
- Товар "Подарочный сертификат" (XML_ID: `GIFT_CERTIFICATE_ONLINE`) отображался в списках товаров каталога вместе с обычными ювелирными изделиями.
- Сертификат является виртуальным товаром и должен приобретаться через специальную форму на странице сертификатов, а не через стандартный каталог.
- Показ сертификата в общих списках товаров создавал путаницу для пользователей.

## Key Decisions
- Добавлен фильтр исключения сертификата в компоненте `app:catalog.full` на уровне метода `executeComponent()`.
- Фильтр применяется ко всем страницам каталога (категории, подкатегории, модели, поиск, коллекции, секции) через глобальную переменную фильтра `FILTER_NAME`.
- Используется константа `CatalogCertificate::CERTIFICATE_PRODUCT_XML_ID` для унификации и избежания хардкода.
- Фильтр применяется после всех `prepareData*` методов, но до `setChain()`, чтобы исключение работало на всех типах страниц.

## Code Touchpoints
- `app/local/components/app/catalog.full/class.php`:
  - Добавлен импорт `\App\Catalog\Certificate as CatalogCertificate` (строка 7).
  - Добавлен фильтр исключения (строки 115-116): `global ${$this->arParams['FILTER_NAME']}; ${$this->arParams['FILTER_NAME']}['!XML_ID'] = CatalogCertificate::CERTIFICATE_PRODUCT_XML_ID;`
  - Фильтр применяется в методе `executeComponent()` после всех `prepareData*` методов.

## Verification
- Товар-сертификат не отображается в списках товаров на страницах категорий, подкатегорий, моделей, поиска, коллекций и секций.
- Сертификат остается доступным для покупки через специальную форму на странице сертификатов.
- Фильтр применяется универсально ко всем типам страниц каталога через единую точку в `executeComponent()`.

## Follow-ups
- При необходимости проверить, что сертификат не попадает в другие компоненты каталога (например, похожие товары, просмотренные товары).
