# Task Memory Card
Date: 2026-01-23
Task Key: 1296
Title: Фикс запроса карты при регистрации

## Context
- При регистрации нового пользователя через метод `Auth::loginOrRegister()` происходит запрос карты лояльности через `Trimiata1c::getDiscountCards([$phone])`.
- Метод `updateDiscountCard` ожидает три параметра: `$user`, `$userData`, `$cardData`, где `$userData` — данные клиента без массива карт, а `$cardData` — данные конкретной карты.
- Раньше в `Auth.php` передавался весь объект клиента (`$card`) как второй параметр, что приводило к некорректной обработке данных карты.

## Key Decisions
- Разделить данные клиента и карты перед передачей в `updateDiscountCard`.
- Извлечь данные клиента: `$userData = $client; unset($userData['cards']);` — удалить массив карт из данных клиента.
- Извлечь последнюю карту из массива: `$cardData = $client['cards'][array_key_last($client['cards'])];` — использовать последнюю карту из массива карт клиента.
- Передать разделенные данные: `updateDiscountCard($user, $userData, $cardData)` вместо `updateDiscountCard($user, $card)`.

## Code Touchpoints
- `app/local/php_interface/lib/app/User/Auth/Auth.php`:
  - Метод `loginOrRegister()` (строки 67-75): разделение данных клиента и карты перед вызовом `updateDiscountCard`.
  - Изменена переменная `$card` на `$client` для ясности (строка 67).
  - Добавлено извлечение `$userData` без массива карт (строки 70-71).
  - Добавлено извлечение `$cardData` как последней карты из массива (строка 73).
  - Обновлен вызов `updateDiscountCard` с тремя параметрами (строка 75).

## Verification
- При регистрации нового пользователя данные карты лояльности корректно сохраняются в профиле пользователя.
- Метод `updateDiscountCard` получает корректно разделенные данные клиента и карты.
- Информация о карте лояльности (тип, номер, GUID, скидка) корректно записывается в свойства пользователя.
