# 1250: Вставки на детальной

**Дата:** 2026-01-05  
**Задача:** 1250

## Context

Требовалось реализовать блок "Вставки" на детальной странице товара для отображения информации о камнях (вставках) в изделиях. Блок должен был соответствовать дизайну из Figma, отображать данные из Highload-блока InsertTable и автоматически обновляться при смене веса товара без перезагрузки страницы.

## Key Decisions

1. **Верстка блока из Figma** — реализована в методе `TemplateHelper::getInsertsHtml()` с адаптацией под реальную структуру JSON-данных
2. **Интеграция в детальную страницу** — блок выводится в табе "Характеристики" через `result_modifier.php`
3. **Адаптация структуры данных** — маппинг полей JSON (kamen, forma_ogranki, gruppa_cveta, gruppa_defecta, ves, kolichestvo, ed_izm, cvet_kamnya) на отображаемые характеристики
4. **AJAX-обновление при смене веса** — создан API endpoint и добавлена логика обновления блока аналогично блоку доставки
5. **Оптимизация запроса** — получение `PROPERTY_INSERT` напрямую в выборке через `GetNext(false, false)` вместо `GetProperties()`

## Code Touchpoints

### Backend
- `app/local/php_interface/lib/app/Template/Helper.php` — метод `getInsertsHtml($insert)`:
  - Получает данные из Highload-блока `InsertTable` по `UF_XML_ID`
  - Парсит JSON из `UF_INSERTS_INFO`
  - Рендерит HTML с заголовком "Вставки" и карточками камней
  - Отображает иконки, названия, вес, количество, характеристики (форма, цветность, огранка, цвет, чистота)
  - Поддерживает collapse/expand для детальной информации
- `app/local/templates/trimiata/components/app/catalog.element/main/result_modifier.php`:
  - Интеграция блока вставок в таб "Характеристики"
  - Вызов `TemplateHelper::getInsertsHtml()` для выбранного оффера
- `app/api/product/inserts.php` (новый) — API endpoint для получения HTML вставок по XML_ID оффера:
  - Получает оффер из `APP_IBLOCK_CATALOG_SKU` по XML_ID
  - Извлекает `PROPERTY_INSERT_VALUE` напрямую из выборки
  - Генерирует HTML через `TemplateHelper::getInsertsHtml($insert)`

### Frontend
- `app/local/changes/template/src/styles/template/layout/pages/product/product-inserts/product-inserts.scss` (новый) — стили для блока вставок:
  - Градиентный фон заголовка
  - Стили карточек камней с границами и скруглениями
  - Адаптивная верстка
- `app/local/templates/trimiata/js/AppApi.js` — добавлен метод `inserts()` для вызова API
- `app/local/templates/trimiata/js/App.js` — добавлен метод `getProductInserts(productXmlId)`
- `app/local/templates/trimiata/js/AppBasket.js`:
  - Добавлен селектор `detailInserts: '[data-role=product-inserts]'` в `elements`
  - В обработчике `product-change-offer` добавлено обновление блока вставок через AJAX
  - Замена содержимого контейнера при успешном ответе

## Gotchas

- **Структура JSON-данных** — данные приходят в формате массива объектов или одного объекта, требуется нормализация через проверку `isset($insertsInfo[0])`
- **Маппинг полей** — реальные поля JSON отличаются от ожидаемых: `kamen` вместо `name`, `forma_ogranki` вместо `form`, `gruppa_cveta` вместо `colorness`, `gruppa_defecta` вместо `clarity`
- **Единицы измерения** — вес может быть в граммах (`гр.`) или каратах (`ct.`), требуется обработка через `ed_izm`
- **Блок вставок может отсутствовать** — если у оффера нет вставок, проверка `insertsContainer` обязательна
- **HTML структура** — HTML вставок включает заголовок "Вставки" и сам блок `product-inserts`, при AJAX-обновлении извлекаем только содержимое блока
- **Оптимизация запроса** — использование `GetNext(false, false)` вместо `GetNextElement()` + `GetProperties()` более эффективно (один запрос вместо двух)

## Verification

1. Открыть детальную страницу товара с вставками
2. Проверить отображение блока "Вставки" в табе "Характеристики"
3. Проверить корректность отображения всех характеристик камней (название, вес, количество, форма, цветность, огранка, цвет, чистота)
4. Проверить работу collapse/expand для детальной информации
5. Кликнуть на элемент с `data-role="product-change-offer"` для смены веса
6. Проверить, что блок вставок обновляется при смене веса без перезагрузки страницы
7. Проверить, что если у нового оффера нет вставок, блок корректно обрабатывается
