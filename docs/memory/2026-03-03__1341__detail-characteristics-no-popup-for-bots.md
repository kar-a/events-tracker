# 1341: Характеристики на детальной без попапа для поисковых роботов

## Context
На детальной странице товара вкладка «Характеристики» выводилась через offcanvas (боковая панель): контент подгружался в разметку панели, открываемой по клику (data-bs-toggle="offcanvas"). Поисковые роботы не выполняют JavaScript и не «открывают» панели, поэтому блок характеристик не попадал в индексируемый HTML страницы — только заголовок вкладки был виден в разметке. Это ухудшало SEO: характеристики товара не участвовали в индексации и в сниппетах поисковиков.

## Причинно-следственные связи
- **Проблема:** Один шаблон для всех: вкладка PROPERTIES всегда рендерилась через `TemplateHelper::getAsideTemplate()` (offcanvas). Контент характеристик формируется в result_modifier (DISPLAY_PROPERTIES, вставки) и попадает в $tab['CONTENT']; при offcanvas этот контент уходит в разметку панели, которая по умолчанию скрыта и не раскрывается роботами.
- **Решение:** Разделить вывод по типу клиента: при запросе от робота (`Router::getInstance()->isBot()`) выводить контент вкладки «Характеристики» так же, как остальные вкладки — в блоке `<div class="product_tab__section collapse">` с `$tab['CONTENT']` внутри страницы. Для обычных пользователей оставить offcanvas.
- **Следствие:** Роботы получают полный HTML страницы с характеристиками в теле документа; пользователи по-прежнему открывают характеристики в боковой панели. Один файл изменён — шаблон детальной.

## Key Decisions
- Использовать существующий `Router::getInstance()->isBot()` (конфигурация ботов в `app/local/php_interface/config/uri.php`) для определения робота; не вводить отдельный флаг или cookie.
- Для ботов использовать ту же ветку вывода, что и для вкладок DESCRIPTION/DELIVERY и т.д. (collapse-блок с CONTENT), чтобы характеристики были в DOM без зависимости от JS.
- COMPACT_MODE и логика остальных вкладок не меняются; правка только в условии выбора между getAsideTemplate и inline-блоком для вкладки PROPERTIES.

## Code Touchpoints
- `app/local/templates/trimiata/components/app/catalog.element/main/template.php` — цикл по $arResult['TABS']: для вкладки с CODE === 'PROPERTIES' при `$arParams['COMPACT_MODE'] !== 'Y' && !Router::getInstance()->isBot()` вызов getAsideTemplate($config); иначе вывод `<div class="product_tab__section collapse" id="..."><?= $tab['CONTENT'] ?></div>`.
- Формирование контента характеристик без изменений: `app/local/templates/trimiata/components/app/catalog.element/main/result_modifier.php` (секция PROPERTIES, DISPLAY_PROPERTIES, вставки).

## Gotchas
- Заголовок вкладки «Характеристики» для ботов по-прежнему имеет data-bs-target на offcanvas; роботы не кликают, поэтому неважно. Важно лишь, что контент дублируется во второй ветке (collapse) и присутствует в HTML.
- При добавлении других вкладок в offcanvas для пользователей — учитывать, нужна ли аналогичная ветка «для ботов» (контент в HTML).

## Verification
- Запрос детальной страницы с User-Agent поискового бота (например, Googlebot) — в HTML присутствует блок с id="tab_properties" и внутри список характеристик (product__attributes, product_attribute).
- Обычный пользователь: клик по «Характеристики» открывает боковую панель, контент совпадает с тем, что виден роботу в collapse-блоке.

## Follow-ups
- При необходимости проверить индексацию через «Просмотр как Google» или инструменты вебмастера; при появлении других JS-зависимых блоков на детальной — оценить вывод альтернативной версии для ботов.
