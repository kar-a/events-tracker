# 1154 — Страница «О нас»

Дата: 2025-09-25

## Контекст
Создание и доработка контентной страницы `app/company/index.php` на основе черновиков HTML из `app/local/changes/template/pages/company/src/`. Цели: локализация ассетов (SVG/PNG), единые правила юнитов (rem), консистентная структура и подготовка к полуавтоматической генерации верстки.

## Что сделано
- Перенесён и структурирован контент страницы «О нас» в `app/company/index.php`.
- Блоки переведены на PHP‑массивы + foreach:
  - `company_assortment__grid` (ассортимент)
  - `company_advantages__grid` (наши преимущества)
  - `company_locations__grid` (где нас найти)
- Локализованы внешние изображения Anima (SVG/PNG):
  - SVG — предпочтительно inline в разметке; иконки/изображения — локальные пути `SITE_TEMPLATE_PATH/bundle/img/company/*`.
- Компонент `app:banners`:
  - В `class.php` добавлен `DETAIL_TEXT` в выборку/результат.
  - В шаблоне `slider/template.php` `DETAIL_TEXT` выводится отдельным блоком над `<picture>`.
- SCSS:
  - Перевод px→rem в `section.scss`, `search-form.scss`.
  - Подключён `company.about.scss` в `pages.scss`.

## Инварианты/правила
- Единицы: размеры в rem; px допустим для `border-width`, `box-shadow`, тонких линий.
- Классы: snake_case, BEM‑подобная структура блоков страницы.
- Ассеты: все внешние ссылки заменять на локальные; SVG — inline для иконок, для картинок — локальные файлы в bundle.
- HTML скелет: `.container > .section > .page(.page_overview) > [aside].page_sidebar__list + .page-overview > .page__article*`.
- Баннеры: если `DETAIL_TEXT` заполнен — выводить над `<picture>`.

## Files → Effects → Intent
- Files → `app/company/index.php`, `app/local/components/app/banners/class.php`, `app/local/templates/trimiata/components/app/banners/slider/template.php`, SCSS файлы (`section.scss`, `search-form.scss`, `company.about.scss`, `pages.scss`)
- Effects → Поддерживаемая верстка через PHP‑массивы; локальные ассеты; гибкость баннеров; единые юниты.
- Intent → Снизить регрессии, ускорить правки и автоматизацию верстки.

## Проверки/QA
- Нет внешних ссылок на ассеты (grep по `https://c.animaapp.com`).
- Изображения рендерятся из `SITE_TEMPLATE_PATH/bundle/img/company/*`.
- `<img>` использует только `alt` (без дублирующего `title`).
- Блоки перечислений рендерятся из массивов (`foreach`).

## Заметки
- При дальнейшем развитии: собрать фронт `npm run build && npm run postbuild`, убедиться, что изображения попали в bundle; paths проверить.

## 1154: Страница «О нас» — перевёрстка и локализация ассетов

Дата: 2025-09-25

### Context
- Требовалось переверстать страницу `/company/` («О нас») под единые правила шаблона: локальные ассеты, inline‑SVG, блоки из PHP‑массивов без ORM/логики в шаблоне, размеры в rem.

### Key Decisions
- Разметка блоков как данные: массивы PHP с `title`, `description`, `image/svg`, итерация `foreach`.
- Ассеты: все внешние изображения и SVG заменены на локальные; SVG — inline для контроля цвета/размера.
- Единицы: `rem` вместо `px` (кроме border‑width/box‑shadow); классы — `snake_case`.
- Страницы подключают page‑стили через `pages.scss` (компоновка в bundle).

### Code Touchpoints
- `app/company/index.php` — основная разметка страницы «О нас» (блоки values/symbol/assortment/advantages/locations/team), данные через массивы.
- Стили: `app/local/changes/template/src/styles/template/layout/pages/company/company.about.scss` (подключено в pages.scss), изображения под `templates/trimiata/bundle/img/company/*`.
- JS: переиспользование существующих модулей (инициализация слайдеров через `App.js`/`$.initialize` при необходимости), без новых зависимостей.

### Gotchas
- В шаблоне нет вызовов ORM/хелперов, только подготовленные данные и верстка.
- Все пути ассетов — относительные к `SITE_TEMPLATE_PATH/bundle/...`.
- Минимизировать инлайновые стили; использовать общие utility‑классы проекта.
