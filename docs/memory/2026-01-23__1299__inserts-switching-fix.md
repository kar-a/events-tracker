# Task Memory Card

Date: 2026-01-23
Task Key: 1299
Title: Фикс переключения вставок

## Context
При переключении веса (offer) товара на детальной странице обновлялись цена и информация о доставке, но блок с информацией о вставках оставался без изменений. Это приводило к тому, что пользователь видел устаревшую информацию о камнях (вставках) в изделии при выборе другого веса.

## Problem
При смене веса товара на детальной странице:
- Обновлялись цена и информация о доставке через AJAX
- Блок вставок (`[data-role=product-inserts]`) не обновлялся
- Пользователь видел информацию о вставках для предыдущего веса, что могло вводить в заблуждение

## Solution
Добавлена логика AJAX-обновления блока вставок при переключении веса аналогично блоку доставки:
- Добавлен метод `inserts()` в `AppApi.js` для вызова API endpoint `product/inserts.php`
- Добавлен метод `getProductInserts()` в `App.js` для получения информации о вставках
- В обработчике смены веса (`AppBasket.js`) добавлена логика обновления контейнера вставок
- При переключении веса блок вставок обновляется через AJAX с новым HTML-содержимым

## Key Decisions
1. **Использование существующего API endpoint** — используется уже созданный `app/api/product/inserts.php` (из задачи 1250), который возвращает HTML блока вставок по XML_ID оффера
2. **Аналогичная логика с блоком доставки** — обновление блока вставок реализовано по аналогии с блоком доставки для единообразия кода
3. **Простое обновление содержимого** — замена содержимого контейнера `[data-role=product-inserts]` без сложной анимации (в отличие от блока доставки)
4. **Проверка наличия контейнера** — обновление выполняется только если контейнер вставок найден в DOM

## Code Touchpoints

### Frontend
- `app/local/changes/template/src/js/AppApi.js` — добавлен метод `inserts(data, method = 'POST')` для вызова API endpoint `product/inserts.php` (строка 177-179)
- `app/local/changes/template/src/js/App.js` — добавлен метод `getProductInserts(productXmlId)` для получения информации о вставках (строки 2059-2064)
- `app/local/changes/template/src/js/app/AppBasket.js`:
  - Добавлен селектор `detailInserts: '[data-role=product-inserts]'` в объект `elements` (строка 13)
  - Добавлена переменная `insertsContainer` для получения контейнера вставок (строка 258)
  - Добавлена логика обновления блока вставок при переключении веса (строки 354-376):
    - Проверка наличия `elementXmlId`, `insertsContainer` и `productData`
    - Вызов `app.getProductInserts(elementXmlId)` для получения HTML вставок
    - Замена содержимого контейнера `insertsContainer` новым HTML из ответа API

## Effects
- Блок вставок автоматически обновляется при переключении веса товара на детальной странице
- Пользователь видит актуальную информацию о камнях (вставках) в изделии для выбранного веса
- Улучшен пользовательский опыт просмотра товаров на детальной странице
- Единообразное поведение обновления информации при смене веса (цена, доставка, вставки)

## Intent
Исправить проблему с отображением устаревшей информации о вставках при переключении веса товара на детальной странице. Обеспечить актуальность информации о составе изделия при выборе другого веса, что важно для принятия решения о покупке и улучшает пользовательский опыт.
