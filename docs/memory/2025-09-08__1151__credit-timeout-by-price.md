Date: 2025-09-08
Task: 1151 — Увеличение времени на рассрочку (период охлаждения)

Context
- Крон `app/local/cron/orders/cancel_unpaid.php` раз в минуту отменяет неоплаченные заказы.
- Порог для онлайн‑оплаты картой — 10 минут (`Exchange::MAX_MINUTES_STAY_UNPAID`). Для рассрочки — 90 минут (`MAX_MINUTES_STAY_UNPAID_CREDIT`).
- Требование: для рассрочки добавить «период охлаждения» по сумме заказа: <50 000 → base; 50 000–200 000 → base+4 ч; >200 000 → base+48 ч.

Key Decisions
- Инкапсулировать расчёт таймаута в новую функцию `Exchange::getCreditUnpaidTimeoutMinutesByPrice($price)`.
- Сохранить текущую выборку заказов в кроне и добавить ранний выход (continue) для кредитных заказов, если увеличенный таймаут ещё не истёк. Это минимальная правка без усложнения ORM‑фильтра.

Code Touchpoints
- `app/local/php_interface/lib/app/Exchange/Exchange.php`
  - Добавлена `public static function getCreditUnpaidTimeoutMinutesByPrice($orderPrice): int`.
- `app/local/cron/orders/cancel_unpaid.php`
  - Перед отменой, для платежей из `Order\Helper::CREDIT_PAY_SYSTEMS`, вычисляется увеличенный таймаут по `getPrice()` и при его неистечении отмена пропускается.

Gotchas
- Фильтр по дате для кредитов остаётся на «базовом» пороге (90 мин), поэтому часть заказов будет попадать в выборку раньше, но отмена не произойдёт из‑за дополнительной проверки — допустимо, нагрузка минимальна.
- Границы интервалов: 50 000 и 200 000 включены в средний диапазон (4 часа), >200 000 — старший.

Verification
- Заказ с рассрочкой на 30 000 → отменяется через ~90 мин.
- На 50 000–200 000 → ~90 мин + 4 ч.
- На 250 000 → ~90 мин + 48 ч.
- Заказы по карте — без изменений (10 мин). Поведение при оплаченных/отменённых заказах неизменно.

Follow-ups
- При желании можно «поднять» логику в фильтр ORM тремя OR‑ветками по PRICE, чтобы не выбирать ранее срока; пока не требуется.
