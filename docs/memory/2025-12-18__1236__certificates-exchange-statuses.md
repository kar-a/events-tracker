# Task Memory Card

Date: 2025-12-18
Task Key: 1236
Title: Обмен статусами сертификатов

## Context
- Проблема: необходимо реализовать обмен сертификатами по типу обмена товарами - получать полный список сертификатов из 1C, добавлять недостающие на сайт, деактивировать лишние
- Ограничения: API 1C возвращает список сертификатов с полями number, purchase_date, expiration_date, count, sum
- Базовое поведение: сертификаты хранятся в HL блоке CertificatesTable, для них создаются купоны в правилах работы с корзиной

## Key Decisions
- Использование одного массива $data вместо множества переменных → Упрощает код и уменьшает количество промежуточных переменных → Улучшает читаемость и поддерживаемость
- Выполнение операций сразу в цикле вместо сбора в массивы → Операции выполняются по мере обработки данных → Улучшает производительность и упрощает логику
- Добавление поля UF_EXPIRATION_DATE в HL блок → Позволяет хранить и проверять дату окончания действия сертификата → Обеспечивает корректную работу с истекшими сертификатами
- Автоматическая активация/деактивация купонов при изменении статуса сертификатов → Синхронизирует состояние купонов со статусом сертификатов → Гарантирует корректную работу скидок по сертификатам
- Проверка даты окончания действия при применении скидки → Предотвращает применение скидки по истекшим сертификатам → Обеспечивает корректную работу системы скидок

## Gotchas (Pitfalls)
- `PaySystemManager::getListWithRestrictions($payment)` требует установки полей SUM и CURRENCY для корректной проверки ограничений по цене
- При обработке даты окончания действия нужно проверять формат даты из 1C (Y-m-d\TH:i:s) и обрабатывать некорректные значения (0001-01-01T00:00:00)
- При создании купонов нужно использовать реальную дату окончания действия из 1C, а не фиксированные 2 года
- Функция ulogging() может не создать файл, если папка не существует - необходимо создавать папки рекурсивно
- При деактивации сертификатов нужно также деактивировать связанные купоны
