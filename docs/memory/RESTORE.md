# Техника восстановления карточек памяти задачи

Цель: быстро восстановить контекст задачи (Files → Effects → Intent) по истории репозитория и синхронизировать `docs/memory/*`.

## Алгоритм
1) Поиск задачи
   - Если карточка есть в `docs/memory/INDEX.md` → открыть и проверить актуальность.
   - Если карточки нет → найти коммиты: `git log --date=short --pretty=format:"%ad %h %s" --grep="\b<TASK>\b"`.

2) Извлечение изменений
   - Для каждого релевантного коммита: `git show --name-only <hash>`.
   - Зафиксировать перечень файлов и предполагаемые зоны влияния (компоненты, шаблоны, JS, helpers, конфиги, docs).

3) Причинно‑следственные связи
   - Сопоставить изменения кода с бизнес‑поведением: URL/фильтры, кеш/инвалидация, SEO/meta, AJAX‑потоки.
   - Выписать инварианты (канонический порядок, исключения из SMART_FILTER_PATH, нормализация URL и т.п.).

4) Карточка памяти
   - Создать `docs/memory/YYYY-MM-DD__<task>__<slug>.md` по шаблону (Context, Key Decisions, Code Touchpoints, Gotchas, Verification, Follow-ups).
   - Дату взять из git log (первый/ключевой коммит задачи).
   - Добавить запись в `docs/memory/INDEX.md`.

5) Санитария
   - Удалить/переименовать дубликаты карточек, чтобы имя файла совпадало с датой в индексе.
   - Проверить, что все ссылки из `INDEX.md` ведут на существующие файлы.

## Подсказки
- «Зоны истины»: `ajax_before.php`/`ajax_after.php`, `AppSmartFilter.js`, `Catalog/Helper.php`, `Seo/Seo.php`, конфиг `config/catalog.php`.
- При несоответствиях дат — нормализовать под историю git (индекс ↔ имя файла ↔ поле Date).
- Фиксировать выделенные принципы (например, snake_case для алиасов, исключение SUBCATEGORY из SMART_FILTER_PATH, разделитель `-i-`).
