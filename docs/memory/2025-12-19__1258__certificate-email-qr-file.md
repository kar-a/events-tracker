# 1258: QR в письмах на gmail

**Date**: 2025-12-19  
**Status**: Completed

## Context
Gmail фильтрует base64-изображения в письмах, что приводит к отсутствию QR-кодов в письмах с сертификатами. QR-коды были реализованы в задаче 1255 и сохранялись как base64-строка в поле `CERTIFICATE_QR`.

## Key Decisions
- QR-код сохраняется как отдельный файл PNG в директории `/upload/certificates/qr/` вместо base64-строки → Gmail и другие почтовые клиенты корректно отображают обычные URL изображений → QR-коды доступны всем получателям
- Используется полный URL с `APP_MAIN_HOST` для изображения → гарантирует доступность файла из письма → работает независимо от контекста открытия письма
- Имя файла формируется из номера сертификата с заменой спецсимволов на подчеркивания → безопасно для файловой системы → уникальность имени обеспечивается уникальностью номера сертификата
- Директория создается автоматически при первом использовании → не требует ручной подготовки → упрощает развертывание

## Code Touchpoints
- `app/local/php_interface/lib/app/Order/Helper.php`
  - Метод `prepareCertificateData()`: изменена логика сохранения QR-кода. Вместо base64-строки QR-код сохраняется как файл PNG через `file_put_contents()` в директории `/upload/certificates/qr/`. Директория создается автоматически через `mkdir()` с правами 0755. Имя файла формируется из номера сертификата через `preg_replace('/[^a-zA-Z0-9_-]/', '_', ...)` с добавлением расширения `.png`. В поле `CERTIFICATE_QR` возвращается HTML-тег `<img>` с полным URL через `APP_MAIN_HOST . '/upload/certificates/qr/' . $fileName`

## Gotchas (Pitfalls)
- Если номер сертификата содержит много спецсимволов, после замены имена файлов могут конфликтовать → необходимо убедиться, что номера сертификатов уникальны и не содержат только спецсимволы
- Файлы QR-кодов накапливаются в директории `/upload/certificates/qr/` → может потребоваться периодическая очистка старых файлов → рекомендуется настроить задачу очистки для истекших сертификатов
- Директория должна быть доступна через веб-сервер → необходимо убедиться, что nginx/апач настроены на обслуживание `/upload/` → в проекте это уже настроено
- При изменении `APP_MAIN_HOST` в конфигурации старые письма будут содержать неверные URL → рекомендуется использовать редирект или хранить файлы на CDN
