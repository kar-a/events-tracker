# 1346: Значения фильтров категорий в sitemap

Date: 2026-01-27
Task Key: 1346
Title: Значения фильтров категорий в sitemap

## Context
- В sitemap нужно было добавить URL страниц фильтров не только «глобально» (от корня каталога), но и в разрезе каждой категории, чтобы в карту попадали ссылки вида `/catalog/koltsa/dizayn_kolca-xxx/` и т.п.
- Ранее getFilterLinks() не принимал категорию, учитывались только свойства DIAMETER и SIZE, а комбинации строились через getPropsCombinations(); ссылки формировались от FILTER_BASE без префикса категории.
- Дополнительные ссылки каталога (фильтры, разделы, поиск) добавлялись при условии `$iblockId != APP_IBLOCK_CATALOG`, из‑за чего для инфоблока каталога они не попадали в sitemap (логическая ошибка).

## Причинно-следственные связи
- **Причина 1:** Отсутствие параметра категории в getFilterLinks() и единый базовый путь — в sitemap не было URL фильтров, привязанных к категории.
- **Причина 2:** Ограниченный набор свойств (только DIAMETER, SIZE) и комбинаторный перебор — не все релевантные страницы фильтров попадали в карту.
- **Причина 3:** Условие `!= APP_IBLOCK_CATALOG` — блок «Дополнительные ссылки для каталога» выполнялся для всех инфоблоков, кроме каталога, поэтому ссылки на фильтры и разделы каталога никогда не записывались.
- **Решение:** Ввести параметр getFilterLinks($category = ''); при $category добавлять в фильтр выборки PROPERTY_CATEGORY и использовать base = FILTER_BASE . $category . '/'. Расширить список свойств; строить ссылки по группировке по каждому свойству. В getSectionLinks() для каждой категории вызывать getFilterLinks($category['CODE']) и мержить результаты. Заменить условие на iblockId == APP_IBLOCK_CATALOG.
- **Следствие:** В sitemap попадают URL фильтров по категориям; полный набор свойств даёт больше релевантных страниц; доп. ссылки каталога снова записываются при генерации.

## Key Decisions
- getFilterLinks($category = '') — при пустой категории сохраняется прежнее поведение (корневой путь), вызов из блока «Дополнительные ссылки» без аргумента даёт «глобальные» ссылки фильтров; getSectionLinks() добавляет ссылки в контексте каждой категории.
- Список свойств зафиксирован: DIAMETER, SIZE, MATERIAL, BRAIDING, LOCK, PROCESSING, DESIGN_RING, DESIGN_PENDANT, INSERT — соответствуют фильтруемым свойствам каталога и getLinkByParams.
- Группировка по одному свойству за итерацию (без getPropsCombinations) — перебор комбинаций заменён на группировку по PROPERTY_<code> с фильтром по активным/доступным товарам с ценой; для каждой группы формируется link через CatalogHelper::getLinkByParams($base, $linkParams, true).

## Code Touchpoints
- `app/local/php_interface/lib/app/Seo/Sitemap.php`:
  - getFilterLinks($category = ''): параметр категории; фильтр по PROPERTY_CATEGORY при $category; расширенный $props; base = FILTER_BASE . $category . '/' при $category; группировка по одному свойству в цикле.
  - getSectionLinks(): после добавления URL категории вызов getFilterLinks($category['CODE']) и array_merge результатов.
  - Блок «Дополнительные ссылки для каталога»: условие изменено с `$iblockId != APP_IBLOCK_CATALOG` на `$iblockId == APP_IBLOCK_CATALOG`.

## Gotchas
- Дедупликация URL (1342) выполняется при записи через addIBlockEntry; возможны пересечения между getFilterLinks() (корень) и ссылками из getSectionLinks() (категории) — реестр $NS['SITEMAP_URLS'] устраняет дубли.
- getFilterLinks() для подкатегорий не вызывается — только для категорий верхнего уровня (category['CODE']). При необходимости расширить на подкатегории — передавать путь/код подкатегории и учитывать в base.

## Verification

- Убедиться, что в сгенерированном XML присутствуют URL вида /catalog/{category}/... с параметрами фильтров (свойства из списка).
- Проверить, что количество доп. ссылок каталога (фильтры + разделы + поиск) ненулевое и что в sitemap есть разделы каталога и страницы поиска.

## Follow-ups
- При добавлении новых фильтруемых свойств каталога — добавить код в массив $props в getFilterLinks().
- При необходимости включать в sitemap фильтры в контексте подкатегорий — расширить getSectionLinks() вызовами getFilterLinks с путём/кодом подкатегории и согласовать base с getLinkByParams.
