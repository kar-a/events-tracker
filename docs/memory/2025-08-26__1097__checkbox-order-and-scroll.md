Task: 1097 — Убрать выставление вверх выбранных чекбоксов и сохранять прокрутку

Context
- В смарт‑фильтре выбранные значения чекбоксов поднимались в начало группы после применения фильтра, что сбивало привычный порядок и контекст пользователя.
- При AJAX‑обновлении фильтра прокрутка группы чекбоксов сбрасывалась в начало.

Key Decisions
- Шаблон: убрать принудительную сортировку выбранных значений — удалён `usort` перед выводом списка чекбоксов.
- JS: сохранить/восстановить `scrollTop` для контейнеров чекбокс‑групп при замене HTML после AJAX‑ответа.

Code Touchpoints
- `app/local/templates/trimiata/components/app/catalog.smart.filter/main/template.php` — удалена сортировка `usort` для `DISPLAY_TYPE='F'`.
- `app/local/templates/trimiata/js/AppSmartFilter.js` — сохранение карты прокруток до перерисовки и восстановление после (`catalog_sidebar__checkbox_group`).

Gotchas
- Идентификация контейнеров для прокрутки: используем `id` блока либо `id` родителя; при отсутствии — временный ключ.
- Не менять исходный порядок значений — сортировка должна управляться бэкендом/настройками, а не состоянием `CHECKED`.

Verification
- Выбор значения (например, `dizayn_podveski=...`) не меняет порядок пунктов — выбранный остаётся на своём месте.
- После AJAX‑обновления полоса прокрутки остаётся на прежней позиции.

Follow-ups
- При необходимости унифицировать `id` контейнеров групп в шаблоне для более надёжного восстановления прокрутки.
