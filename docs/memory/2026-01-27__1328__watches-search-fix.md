# Задача 1328: Часы мужские: фикс поиска

**Дата:** 2026-01-27  
**Задача:** 1328  
**Тип:** Исправление ошибки, Поиск

---

## Контекст

**Проблема:** При вводе запроса "часы мужские" в поиске выводятся все часы из каталога, а не только элементы с алиасом "мужские". При вводе только "мужские" поиск работает корректно.

**Ограничения:** 
- Необходимо сохранить существующую логику поиска
- Фильтр должен применяться только при наличии слова "часы" в запросе
- При запросах без слова "часы" поведение не должно изменяться

**Базовое поведение:** 
- Компонент `app:catalog.search.full` использует `app:search.page` для полнотекстового поиска
- Фильтр передается через глобальную переменную `$arrAddFilter`
- `search.page` использует `$arrAddFilter` для дополнительной фильтрации результатов

---

## Ключевые решения

### Добавление фильтра по категории при наличии слова "часы"

**Решение:** Создать функцию `getQueryBasedCatalogFilter()`, которая парсит запрос по словам и при наличии слова "часы" возвращает фильтр `['=PROPERTY_CATEGORY' => 'chasy']`.

**Обоснование:** 
- Ограничение результатов категорией "chasy" позволяет полнотекстовому поиску корректно фильтровать по остальным словам запроса
- При запросе "часы мужские" фильтр ограничивает результаты только часами, а поиск по "мужские" находит только мужские модели
- При запросе без слова "часы" фильтр не добавляется, поведение остается прежним

**Влияние:** Улучшает точность поиска для запросов с указанием категории товаров.

---

## Code Touchpoints

### `app/local/components/app/catalog.search.full/class.php`

**Метод:** `getQueryBasedCatalogFilter($query)` (новый, строки 453-467)
- Парсит запрос по словам через `preg_split()` с поддержкой UTF-8
- Приводит слова к нижнему регистру через `mb_strtolower()`
- При наличии слова "часы" возвращает `['=PROPERTY_CATEGORY' => 'chasy']`
- Иначе возвращает пустой массив

**Метод:** `getCatalogFilter()` (изменен, строки 472-535)
- Объединяет базовый фильтр `['ACTIVE' => 'Y']` с результатом `getQueryBasedCatalogFilter()`
- Использует `$this->arResult['~QUERY']` (сырой запрос) для парсинга
- Фильтр передается в `app:search.page` через глобальную переменную `$arrAddFilter`

---

## Gotchas (Pitfalls)

1. **Использование `~QUERY` вместо `QUERY`**: Необходимо использовать сырой запрос (`~QUERY`) для парсинга, так как `QUERY` уже экранирован через `htmlspecialcharsex()` и может содержать HTML-сущности.

2. **Парсинг с поддержкой UTF-8**: Использование `preg_split()` с флагом `u` для корректной обработки кириллицы и `\p{P}\p{Z}` для разделителей (пробелы, пунктуация).

3. **Точное совпадение слова**: Проверка `$word === 'часы'` должна быть строгой, чтобы не срабатывать на части слов (например, "часовщик").

4. **Имя свойства фильтра**: Использование `=PROPERTY_CATEGORY` (с `=`) для точного совпадения значения свойства категории.

5. **Не изменять запрос**: Функция только добавляет фильтр, не изменяя сам запрос. Метод `modifyQuery()` удаляет слово "часы" из запроса, но это отдельная логика.

## Verification

### Ручное тестирование

1. **Запрос "часы мужские":**
   - Открыть: `https://trimiata.ru/catalog/search/chasy/часы+мужские/`
   - Ожидается: только мужские часы (аналогично запросу "мужские")
   - Проверить количество результатов (должно быть меньше, чем при запросе "часы")

2. **Запрос "мужские":**
   - Открыть: `https://trimiata.ru/catalog/search/chasy/мужские/`
   - Ожидается: поведение без изменений (17 товаров)
   - Фильтр по категории не должен добавляться

3. **Запрос "часы":**
   - Открыть: `https://trimiata.ru/catalog/search/chasy/часы/`
   - Ожидается: все часы из каталога
   - Фильтр по категории должен добавляться, но это не влияет на результаты (все результаты уже из категории "часы")

4. **Запрос без слова "часы":**
   - Открыть любой другой запрос (например, "кольцо золото")
   - Ожидается: поведение без изменений
   - Фильтр по категории не должен добавляться
