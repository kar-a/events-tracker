## 1157 — Подвиды: убрать повтор категории из значений

Context
- В фильтре подкатегорий значения повторяли название категории (напр.: «Браслеты декоративные жёсткие»).
- По задаче 1099 подвиды уже выделены в отдельные сущности/HL и участвуют в ЧПУ/фильтре.

Key Decisions
- Нормализовать имя подвида на сервере при подготовке данных компонента `app:catalog.smart.filter.categories`.
- Если подвид не равен точному имени категории, удалить ведущий префикс категории + разделитель (пробел/дефис/двоеточие), с учётом регистра.
- Шаблоны не трогаем (источник истины — `arResult`).

Code Touchpoints
- `app/local/components/app/catalog.smart.filter.categories/class.php`
  - Построена карта `$categoryCodeToName` из `arResult['ITEMS']` (код → имя категории).
  - При сборке `$catCodeToSubs` для каждой записи `SubcategoryTable`:
    - `UF_PLURAL` → `$subNameRaw` → `$subName` (trim)
    - если `$subName` != имя категории (без учета регистра), то `preg_replace('/^catName[\s\-\p{Pd}\:]*/iu', '')` → `mb_ucfirst()`
  - Бизнес‑правило 1‑подкатегории сохранено.
- ImShop (1144): `app/local/php_interface/lib/events/yandexmarket/onExportOfferWriteData.php`
  - Для param подвида применена та же нормализация; кэш категорий обогащён `NAME` из дерева для корректного сравнения.

Gotchas
- Регистронезависимая проверка равенства подвида категории.
- Разделители: пробелы, дефисы (включая `\p{Pd}`), двоеточие.
- Не ломать кейс, применяем `mb_ucfirst` в конце, как и раньше.

Verification
- Категория «Браслеты»: «Подвид браслетов» → значения без слова «браслеты».
- Быстрые ссылки под фильтром показывают те же нормализованные названия.
- В YML/ImShop param для подвида без префикса категории.
- Регрессий по URL/чекбоксам нет (логика кодов не менялась).

Follow-ups
- При появлении исключений (нестандартные разделители) расширить паттерн или внести маппинг в конфиг.
