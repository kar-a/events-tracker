## 1202: Статические и динамические подборки в меню

Context
- Ветка 1202 объединяет доработки из 1192 (меню каталога): статические и динамические подборки.
- Цель: согласовать меню с макетами и ускорить рендер, избегая дублирования логики.

Key Decisions
- Динамические коллекции из 1С (HL `CollectionsTable`) + статические specials (бестселлеры/новинки) отдаются единым способом; спецстраницы `/catalog/bestsellery/` и `/catalog/new/` обрабатываются как `collection` с общим FILTER.
- Дополнительные блоки меню переведены на структуру `ADDITIONAL_BLOCKS['categories']` с уровнями `all` и `<XML_ID>`; уровень `all` строится по `PARAMS`, по категориям блоки показываются только при наличии товаров (проверка `CIBlockElement::GetList`, `nTopCount=1`).
- Кэширование результата построения дополнительных блоков через `App\Cache\Helper` (TTL 1 час).
- Для «Подвесок» подвид `podveski-serdtse` поднимается первым (`usort`).
- Конфигурация вынесена в `collections.menu_additional_blocks`, ассеты иконок берутся из bundle.

Code Touchpoints
- Компонент/шаблон: `app/local/components/app/catalog.section.list/class.php`, `app/local/templates/trimiata/components/app/catalog.section.list/header/{template.php,result_modifier.php}`.
- Конфиги: `app/local/php_interface/config/catalog.php` (раздел `collections.menu_additional_blocks`).
- Кэш: `App\Cache\Helper`.

Gotchas
- Проверка наличия товаров обязательна, чтобы не показывать пустые блоки (минимальный запрос с `nTopCount=1`).
- Соблюдать каноничность ссылок для коллекций (единый билдер/шаблон).

Verification
- Меню «Каталог»: видны блоки «Вид золота», «Вставки», specials (Бестселлеры/Новинки); спецстраницы открываются по чистым URL и ведут на общий layout коллекций.
- При смене категории отображаются только релевантные блоки (если есть товары).
- В «Подвесках» «Сердце» отображается первым.

Follow-ups
- Добавить счётчики/метрики кликов по блокам для валидации эффекта.
- Расширить конфиг для дополнительных маркетинговых подборок на уровне категорий.
