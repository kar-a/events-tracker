# Task Memory Card

Task Key: 1222
Title: Правки NiceSelect

## Context
- Проблема: NiceSelect по умолчанию включает функцию поиска (`searchable: true`), что создает поле `.nice-select-search-box` в выпадающем списке. Для простых селектов (например, сортировка каталога) поиск не нужен и усложняет интерфейс.
- Базовое поведение: NiceSelect инициализируется через `$.initialize('[data-type=nice-select]', ...)` в `App.js`, создавая кастомные селекты для всех элементов с атрибутом `data-type=nice-select`.
- Использование: Селекты используются в формах (например, `app:form.result.new`), сортировке каталога (`app:catalog.sorter`), и других местах, где нужна замена стандартных `<select>`.

## Key Decisions
- Решение: Отключить поиск через опцию `searchable: false` при инициализации NiceSelect и скрыть поле поиска в стилях.
  - Рациональное: Для большинства селектов на сайте поиск не нужен, так как количество опций обычно небольшое. Отключение поиска упрощает интерфейс и ускоряет выбор значения.
  - Влияние: Все селекты с `data-type=nice-select` теперь инициализируются без поиска. Поле `.nice-select-search-box` скрыто в стилях для всех селектов и отдельно для селекта сортировки каталога.
- Дополнительно: Добавлен `placeholder: ' '` (пробел) для корректной работы NiceSelect с пустыми значениями.
- Стили: Добавлен `width: 100%` для `.nice-select-dropdown` для корректного отображения выпадающего списка на всю ширину контейнера.

## Gotchas (Pitfalls)
- Важно: Если в будущем понадобится поиск для каких-то селектов, нужно будет добавить отдельную инициализацию с `searchable: true` или использовать другой атрибут (например, `data-type=nice-select-search`).
- Важно: Placeholder `' '` (пробел) используется для корректной работы NiceSelect с пустыми значениями. Пустая строка может вызвать проблемы.
