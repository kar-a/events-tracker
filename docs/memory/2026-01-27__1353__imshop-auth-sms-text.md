# 1353: Замена текста СМС для входа в приложение (ImShop)

Date: 2026-01-27
Task Key: 1353
Title: Замена текста СМС для входа в приложение (ImShop)

## Context
- В авторизации по телефону код отправляется через `Auth::loginOrRegister()`.
- Для источника мобильного приложения (`USER_SOURCE_IMSHOP`) текст СМС был таким же, как для сайта: «...входа на сайт trimiata.ru», что некорректно по смыслу.

## Key Decisions
- Выделен отдельный текст СМС для `USER_SOURCE_IMSHOP`: «Ваш код для входа в приложение Тримиата».
- Для остальных источников оставлен текущий текст про вход на сайт, чтобы не менять поведение веб-канала.
- Источник отправки (`Sms::SOURCE_IMSHOP` / `Sms::SOURCE_SITE`) сохранён без изменений.

## Code Touchpoints
- `app/local/php_interface/lib/app/User/Auth/Auth.php`
  - `Auth::loginOrRegister()` → добавлен выбор текста сообщения по `userSource` перед `Sms::send()`.

## Gotchas (Pitfalls)
- Не менять текст для сайта: это затрагивает текущие пользовательские ожидания веб-формы.
- Не ломать логику `source`, т.к. она используется в SMS-логировании (task 1351).

## Verification
- Вызвать `Auth::loginOrRegister($phone, UserHelper::USER_SOURCE_IMSHOP, ...)` и проверить текст: «Ваш код для входа в приложение Тримиата».
- Вызвать `Auth::loginOrRegister($phone, UserHelper::USER_SOURCE_SITE_BY_SELF, ...)` и проверить, что текст остался «...входа на сайт trimiata.ru».
- Проверить, что отправка идёт с корректным source (`SOURCE_IMSHOP`/`SOURCE_SITE`).

## Follow-ups
- При расширении сценариев авторизации вынести шаблоны SMS в единый конфиг/словарь для централизованного управления формулировками.
