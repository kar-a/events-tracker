# Task Memory Card

Date: 2025-09-18
Task Key: 1094
Title: Конструктор — «всегда показывать» только цвет; размер скрывать при недоступности

## Context
- После 1094 («не скрывать цвет и размеры») пользователи отмечали сумбур при выборе параметров.
- Принято решение упростить: «всегда видимым» оставить только COLOR; SIZE скрывать, если недоступен для текущей конфигурации.

## Key Decisions
- Исключение при удалении недоступных значений применяется только к `COLOR` (SIZE больше не исключение).
- Сохраняем правило удаления свойств с одним значением (кроме COLOR), чтобы не загромождать UI.
- Обновлён поясняющий комментарий рядом с условием.

## Code Touchpoints
- Files → `app/local/php_interface/lib/app/Catalog/Helper.php` → `getProductConstructor()`
  - Условие удаления недоступных значений: `!in_array($propCode, ['COLOR'])` (ранее: `['COLOR','SIZE']`).
  - Комментарий приведён в соответствие: «цвет показываем всегда».
- Шаблон `app/local/templates/trimiata/components/app/catalog.element/main/template.php`
  - Логика рендера SIZE больше не полагается на «всегда показывать»: недоступные размеры не приходят в данные.

## Gotchas (Pitfalls)
- Для свойств, где все значения становятся недоступными, следить за корректностью fallback URL через под‑свойства (не допускать пустых ссылок).
- Проверить, что выбор цвета корректно переключает на доступный SKU и сохраняет прочие выбранные параметры.

## Verification
- Карточка товара с несколькими цветами/размерами: недоступные размеры не отображаются; цветовой ряд виден всегда.
- Клик по цвету переключает на доступную конфигурацию (URL меняется корректно).
- На мобильных и десктопе нет «пустых» рядов размеров; ошибок в консоли нет.
