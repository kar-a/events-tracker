# Unified event contract

## Why this matters
Every event source must converge into one shared schema:
- website JS
- backend events
- Yandex Metrika import
- future mobile app

If not, downstream analytics and recommendations become unreliable.

## Required base fields
- `event_id` — unique event ID
- `source` — `web | metrika | app | backend`
- `event_type`
- `event_time` — UTC normalized timestamp
- `user_key` — primary anonymous/user key
- `session_key`

## Common optional fields
- `product_id`
- `context_type`
- `context_id`
- `page_url`
- `page_type`
- `referrer`
- `device_type`
- `utm_source`
- `utm_medium`
- `utm_campaign`
- `search_query`
- `position`
- `payload`

## Allowed event types
- `product_view`
- `add_to_cart`
- `remove_from_cart`
- `purchase`
- `search`
- `recommendation_show`
- `recommendation_click`

## Important normalization rules
- timestamps must be normalized to UTC
- product IDs must become numeric when possible
- empty strings should become null for nullable fields
- payload should always remain an object/array
- recommendation events must contain `context_id`
- search events must contain `search_query`
- product events must contain `product_id`

## Identity strategy
### Preferred user key
- `metrika:{client_id}`

### Fallback
- site-generated anonymous ID in localStorage/cookie

### Session key
- generated by website session tracker or imported from source data
