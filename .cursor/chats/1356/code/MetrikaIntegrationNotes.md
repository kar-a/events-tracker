# Заметки по интеграции Metrika для текущей PHP-кодовой базы

## Требование к стилю из чата
В проекте уже есть классы интеграции с Яндексом, и интеграция Metrika должна быть выполнена в том же стиле.

Важный точный фрагмент helper-метода, который пользователь явно предоставил:

```php
/**
 * @param $hostType
 * @param $forceCreate
 * @return Service|mixed
 */
private static function getService($hostType = 'webmaster', $forceCreate = false)
{
    static $services;

    if (is_null($services[$hostType]) || $forceCreate) {
        $services[$hostType] = new Service($hostType);
    }

    return $services[$hostType];
}
```

## Практическое правило
Не «улучшать» этот helper в сгенерированном коде, если пользователь явно не попросил рефакторинг.
Сначала реализовать поддержку Metrika в том же стиле.
И только потом, при отдельном запросе, переносить helper-логику в общий Helper.

## Зоны ответственности класса Metrika
- создать запрос логов;
- получить статус запроса;
- скачать часть данных;
- очистить запрос;
- при необходимости получить список запросов.

## Зоны ответственности import worker
- формировать окна дат;
- не брать текущий день;
- поддерживать повторную загрузку последних 3 дней, так как сессии в Metrika финализируются с задержкой;
- архивировать скачанные части;
- нормализовать импортированные строки в единый event schema.
